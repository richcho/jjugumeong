# JJUGUMEONG 개발 지침

## 목표

- Godot 4.x와 GDScript로 개발한다.
- 현재 우선순위는 오류 없이 실행되는 V0.1이다.
- 장기적으로 문명, 세계, 우주 시스템을 확장할 수 있도록 책임을 분리한다.

## 코드 규칙

- Godot 3 API를 사용하지 않는다.
- 지역 변수와 반환형은 가능한 한 명시한다.
- 네이티브 클래스의 메서드/프로퍼티 이름과 사용자 정의 이름이 충돌하지 않게 한다.
- UI, 게임 상태, 저장, 화면 표현을 서로 분리한다.
- 저장 데이터 변경 시 `schema_version`과 마이그레이션을 갱신한다.
- 외부 에셋 없이도 기본 도형으로 실행 가능해야 한다.
- 서버와 온라인 기능은 명시적인 버전 작업 전까지 추가하지 않는다.
- 변경 후 가능한 범위에서 headless 실행과 스크립트 파싱을 검증한다.

## Git 규칙

- 사용자가 기본 Git으로 직접 커밋·푸시를 요청하면 `git status`, `git add`,
  `git commit`, `git push`를 사용한다.
- PR이 명시되지 않은 직접 푸시 작업에는 GitHub CLI나 PR 흐름을 요구하지 않는다.
