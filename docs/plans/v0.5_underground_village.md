# V0.5 작업 계획 — 지하 마을

작성일: 2026-07-20
상태: 계획 (실행 대기)
목표 예산: 2–3h (제안) · 소속 막: 4막 사회 (`roadmap.md` §2 V0.5, `jjugumeong_master_plan.md` §10)
관련 결정 대기 항목: 능동 참여 비중 확대 방향 / 쥐 개체 개성·고유성 확장 / 막별 시간예산 세부 배분
(`jjugumeong_master_plan.md` §13)

## 0. 버전 계획 체크리스트 자기 점검

`jjugumeong_master_plan.md` §11 "시간예산 관리 원칙"이 V0.5부터 요구하는 6항목을
이 문서가 어떻게 만족하는지 명시합니다.

1. 목표 예산·소속 막 선언 — 위 헤더 참고
2. 해결하는 결정 대기 항목 인용 — §8에서 "능동 참여 비중 확대 방향"의 두 축을
   확정 제안하고, "쥐 개체 개성 확장"에 부분 진전을 제안합니다. 두 항목 모두
   실제 구현 완료 전까지는 결정 대기 상태를 유지합니다(기존 버전들과 동일한
   전환 시점 원칙).
3. 기존 컴포넌트 재사용 검토 — §9에서 재사용 목록과 근거를 정리합니다.
4. 완료 조건에 원장 기록 포함 — §13에 명시, `balancing.md` §4 Table A/B에도
   이 문서 작성과 함께 V0.5 행을 채웠습니다.
5. 장기 시대(6막) 아님 확인 — 4막 사회 범위이며 `roadmap.md` §5 부록(장기
   시대)과 무관합니다.
6. 예산 초과 시 컷 우선순위 — §10에서 터널 입구를 1순위 컷 후보로 지정했습니다.

## 1. 목적

쥐구멍은 현재 레벨 숫자와 강화 버튼일 뿐, 실제로 "마을"이라 부를 수 있는
기능과 화면이 없습니다. V0.5는 `건설 → 완공 효과 → 인구 소비 → 오프라인
진행 → 마을 화면`이 연결되는 첫 시설 건설 시스템을 구현해, 쥐구멍이 기능과
화면 모두에서 마을로 변화하게 합니다(`roadmap.md` V0.5 완료 게이트).

## 2. 플레이 흐름

```text
쥐구멍 Lv.5 달성 ("공동 식탁" 목표 도달, get_next_colony_goal() 기존 체인)
  → 다음 보상 카드에 "지하마을: 식량 창고 건설 가능" 표시
  → 플레이어가 3개 건물 중 원하는 순서로 건설 선택
  → 치즈 소비, 건설 대기 시작 (건설쥐 보너스로 단축, 오프라인 중에도 진행)
  → 완공 시 각 건물 고유 효과 적용 + 마을 배경에 건물 실루엣 표시
  → 인구가 무료 한도를 넘으면 소비 시작, 휴식 공간 완공으로 소비 완화
  → 세 건물 모두 완공되면 V0.5 완료, 터널 입구는 V0.6 세계 지도 게이트로 예고
```

## 3. 확정 수치 (안)

| 건물 | ID | 해금 조건 | 건설 비용 | 건설 대기 | 완공 효과 |
| --- | --- | --- | ---: | ---: | --- |
| 식량 창고 | `food_storage` | 쥐구멍 Lv.5 | 치즈 800 | 90초 | 오프라인 생산 상한 4h → 6h |
| 휴식 공간 | `rest_area` | 쥐구멍 Lv.5 | 치즈 800 | 90초 | 인구 소비 1마리당 소비량 50% 감소 |
| 터널 입구 | `tunnel_entrance` | 쥐구멍 Lv.5 | 치즈 1,200 | 150초 | 완공 즉시 현재 운반량 5회분 지급, V0.6 세계 지도 게이트 플래그만 저장(효과는 V0.8에서 사용) |

- 세 건물은 순서 제약 없이 플레이어가 원하는 순서로 건설합니다(의사결정형
  참여, §8 참고).
- 해금 조건은 기존 `get_next_colony_goal()`의 "공동 식탁"(hole_level 5,
  "식사와 휴식 공간") 목표와 정확히 일치하도록 hole_level 5를 재사용합니다.
  새 임계값을 만들지 않습니다.
- Alpha 배수는 생산에만 적용하고 건설 비용에는 적용하지 않습니다(보육실과
  동일 원칙).

## 4. 인구와 소비

- 무료 인구 한도(`VILLAGE_FREE_POPULATION`): 3마리 (역할보드 해금 문턱과
  동일 값을 재사용해 새 기준을 추가하지 않음)
- 초과 인구 1마리당 초당 소비(`VILLAGE_CONSUMPTION_PER_MOUSE`): 치즈 0.2
- 휴식 공간 완공 시 소비량에 0.5배 적용
- **소비는 초당 생산에서 차감하되, 순생산이 음수가 되지 않도록 0을
  하한으로 고정합니다.** 오프라인 생산 계산도 동일 공식을 사용합니다.
  이 하한이 없으면 인구가 늘수록 방치 보상이 줄어드는 역설이 생겨
  `roadmap.md` 완료 게이트("방치 보상과 소비가 플레이를 강제로 되돌리지
  않음")를 위반합니다.
- 소비로 인한 쥐 손실, 굶주림 페널티, 부정적 이벤트는 이번 버전에서
  추가하지 않습니다(§11 제외 범위).

## 5. 오프라인 진행

- 건설 대기와 소비 계산 모두 기존 `TimeManager`의 오프라인 상한 로직을
  재사용합니다. 새 오프라인 시스템을 별도로 만들지 않습니다.
- 건설 완료 판정은 보육실과 동일하게 `ready_unix` 절대 시각으로 저장해
  앱 종료 중에도 진행됩니다.
- 식량 창고 완공 전 오프라인 상한은 기존 4시간을 유지합니다.

## 6. 저장 스키마 10

새 영구 필드:

```text
village_buildings: Dictionary
  food_storage: { built: bool, ready_unix: int }
  rest_area: { built: bool, ready_unix: int }
  tunnel_entrance: { built: bool, ready_unix: int }
```

- 스키마 9 이하 저장은 세 건물 모두 `built = false`, `ready_unix = 0`으로
  이관합니다.
- 건설 중(`ready_unix > 0`, `built = false`) 상태에서 로드하면 남은 시간
  대신 완료 Unix 시각으로 즉시 재계산합니다(보육실과 동일 패턴).

## 7. 화면과 직접 참여

- 하단 `지하마을` 버튼으로 전용 패널(`VillageView` 가칭)을 엽니다.
- 각 건물 버튼은 이름·비용·대기 시간·완공 효과를 한 번에 보여줍니다
  (`game_design.md` §9 "버튼 이름은 결과와 비용을 동시에 설명" 재사용).
- 완공된 건물은 마을 배경에 고유 실루엣으로 표시합니다(외부 에셋 없이도
  기본 도형으로 표현, `jjugumeong_master_plan.md` §9 릴리스 게이트 준수).
- 다음 보상 카드(`GameManager.get_next_reward_summary()`)에 마을 건설
  목표를 추가해, 지역 해금·군락 목표와 같은 자리에서 안내합니다. 별도
  팝업이나 알림을 새로 만들지 않습니다(§8 유도 방법).

## 8. 게임 경험 정합성 원칙 반영

`jjugumeong_master_plan.md` §12 원칙을 이 버전에서 다음과 같이 구체화합니다.

- **연계 원칙**: 건설은 기존 자원 원장(치즈), `EventBus`, 역할보드(건설쥐
  보너스)와 직접 연결됩니다. 별도 메뉴에 고립된 숫자로 남지 않습니다.
- **능동 참여 비중 확대 (§12.1 두 축 확정 제안)**:
  - ① 유도 방법: 다음 보상 카드에서 "지하마을: OO 건설 가능"을 자연스럽게
    제시합니다. 알림성 팝업은 사용하지 않습니다.
  - ② 참여의 방향성: 의사결정형(3개 건물 중 건설 순서를 플레이어가 선택)과
    육성·꾸미기형(완공 결과가 마을 배경에 실루엣으로 누적 표시)을 결합합니다.
  - 이 두 축이 실제로 채워졌으므로, `jjugumeong_master_plan.md` §13의
    "능동 참여 비중 확대 방향" 항목은 구현 완료 후 "구현됨"으로 전환할
    준비가 된 상태입니다(전환은 실제 구현 완료 시점에 수행, 기존 버전들과
    동일 원칙).
- **개체 독립성 원칙 (부분 진전)**: 마을 화면의 생활쥐 표현에 역할별로
  다른 행동을 배정합니다 — 건설쥐는 최근 건설 중인 건물 주변에서 작업
  동작, 채집쥐는 식량 창고 앞에서 정리하는 동작, 탐험쥐는 터널 입구
  앞에서 대기하는 자세. 완전한 개체별 성격까지는 아니므로 결정 대기
  항목은 이번에도 결정 대기로 유지하되, 역할 기반 차별화로 한 단계
  진전시킵니다.
- **이동 목표지점 정확도**: 마을 화면에 새로 배치하는 오브젝트 좌표도
  `jjugumeong_master_plan.md` §12.2의 기존 분기 기준(버그 확인 후 P0
  수정 또는 품질 게이트)을 그대로 따릅니다. 이번 버전에서 새 기준을
  만들지 않습니다.

## 9. 재사용 컴포넌트와 데이터 정합성

새로 만들기 전에 재사용을 검토했습니다(§0 체크리스트 3번).

- 해금 게이트: `hole_level >= N` 패턴을 보육실과 동일하게 재사용 (N=5)
- 비용 스케일링: `_scaled_cost()` 재사용, 건물별 반복 구매는 없으므로
  단일가만 사용
- 건설 시간 단축: 기존 `get_builder_growth_multiplier()`를 보육실
  성장뿐 아니라 건물 건설 대기시간에도 적용 범위를 넓혀 재사용합니다.
  새 역할 보너스 상수를 추가하지 않습니다.
- 오프라인 진행: `TimeManager`의 기존 상한 로직 재사용 (§5)
- 데이터: `data/buildings/initial_buildings.json`에 이미 5개 건물 골격이
  있었지만 `available_version`이 전부 `"0.6"`으로 표기돼 로드맵과
  어긋났습니다. `food_storage`/`rest_area`/`tunnel_entrance`는 이 문서
  작성과 함께 `"0.5"`로 정정했습니다. `training_ground`는 로드맵 어느
  버전에도 명시적으로 배정되지 않아 `"0.6"`을 유지하고 이번 범위에서
  제외합니다(§11).
- 미니게임: 3.3 미니게임 통합 리팩터링(`SequenceActionView`)은 아직
  착수 전이므로, 이번 버전은 새 인터랙션 컴포넌트를 만들지 않고 버튼
  기반 건설 선택만 사용합니다. 통합 컴포넌트가 준비되면 후속 버전에서
  건설 확인 단계에 점 기반 상호작용을 추가하는 것을 고려할 수 있습니다
  (이번 범위 아님).

## 10. 시간예산 추정 (설계 단계, 정밀 실측 불필요)

`docs/plans/scope_review_1day_target.md` 3.1 방법론(예상 생산 공식 + 중간
이정표 후보 + 확정된 메타시스템 수치 근거)을 적용합니다.

- 건설 비용 합계 800+800+1,200=2,800치즈는 지역 해금 비용(공장 10,000)보다
  작아, 이미 진행 중인 자원 축적 속도로는 지역 하나 해금하는 데 걸리는
  시간보다 짧게 소요될 것으로 추정합니다.
- 건설 대기시간 합계는 최대 330초(약 5.5분)이며 오프라인 중에도 진행되므로
  실제 활성 대기 체감은 이보다 짧습니다.
- 세 건물을 세우는 행위 자체는 약 20~40분 내외로 추정하며, 목표 예산
  2~3시간의 나머지는 인구 소비 루프에 적응하며 자원을 다시 모으는
  자연스러운 재순환 시간이 차지할 것으로 추정합니다.
- **예산 초과 시 컷 우선순위**: 추정 또는 이후 부분 플레이 확인 결과
  2~3시간을 크게 초과하면, 터널 입구를 1순위 컷 후보로 지정합니다.
  터널 입구는 현재 버전에서 즉시 효과보다 V0.6 게이트 예고가 주
  목적이라 없어도 V0.5 완료 게이트(마을 기능·화면 전환, 자원·직업
  루프 연결, 소비의 비강제성)를 해치지 않습니다.
- 이 추정치는 `balancing.md` §4 시간예산 원장 Table A/B에도 반영했습니다.

## 11. 제외 범위

- 훈련장(`training_ground`) — 로드맵에 명시적 버전 배정 없음, 후속 대기
- 건물 Lv.2 이상, 건물 파괴·이전
- 소비로 인한 쥐 손실, 굶주림 등 부정적 이벤트(§4에서 명시적으로 배제)
- 터널 입구를 통한 실제 지도 이동·탐험 (`roadmap.md` V0.8 세계 지도와
  세력에서 다룸, 이번에는 게이트 플래그와 1회성 보상만)
- 미니게임 통합 컴포넌트(`SequenceActionView`) 적용 (§9 참고, 3.4 이후
  별도 작업)
- 마을 전용 배경 아트, 전용 스프라이트 (기본 도형 실루엣으로 대체,
  `docs/plans/scope_review_1day_target.md` 3.5 아트 방향 결정과 별개로
  진행하되 결정되면 후속 연결)

## 12. 책임

| 구성 | 책임 |
| --- | --- |
| `GameManager` | 건설 비용·대기·완공 판정, 인구 소비 계산(0 하한 포함), 저장 상태 |
| `VillageView` (신규) | 건물 버튼, 완공 실루엣과 역할별 생활쥐 표현 |
| `GameUI` | 패널 구성, 다음 보상 카드에 마을 목표 연결 |
| `SaveManager` | 스키마 10 마이그레이션 |

UI와 렌더러는 치즈나 인구 수치를 직접 변경하지 않습니다(기존 원칙 유지).

## 13. 완료 조건

- 쥐구멍 Lv.5 미만에서는 건설 패널이 해금 조건과 이유를 표시하고 건설을
  거부합니다.
- 비용이 부족하면 건설이 거부됩니다.
- 세 건물을 플레이어가 원하는 순서로 자유롭게 선택해 건설할 수 있습니다.
