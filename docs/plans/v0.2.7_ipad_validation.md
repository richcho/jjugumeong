# V0.2.7 작업 계획 — iPad 실기기 검증 준비

작성일: 2026-07-19
상태: 자동 검증 완료 · 배포 및 iPad 실기기 검증 대기

## 1. 작업 목적

V0.2.6은 코드와 headless 회귀 검증을 통과했지만, 실제 iPad Safari/PWA에서
검증할 빌드가 현재 소스와 같은 버전인지 자동으로 확인하는 기준이 없습니다.
V0.2 완료 판정을 위해 재현 가능한 Web 릴리스 산출물과 실기기 체크리스트를
준비합니다.

## 2. 이번 범위

1. 제품명, 빌드 버전과 개발 단계를 단일 데이터 파일에서 관리합니다.
2. 게임 UI와 Web 산출물이 같은 빌드 버전을 표시하도록 연결합니다.
3. Web 산출물의 viewport, PWA manifest, 서비스 워커, gzip WASM과 필수 파일을
   검사하는 자동 검증 스크립트를 추가합니다.
4. Godot Web export, iPad용 산출물 준비와 Vinext 프로덕션 빌드를 수행합니다.
5. 대표 iPad 가로·세로에서 사람이 확인할 항목과 결과 기록 형식을 문서화합니다.
6. 자동 검증 결과를 changelog와 기준 문서에 기록합니다.

## 3. 제외 범위

- 실제 iPad를 조작한 것으로 간주하거나 결과를 추정하는 것
- V0.3 지역 선택, 위험 사건, 직업과 도감
- 저장 필드 및 스키마 변경
- 게임 밸런스 변경
- 검증 편의를 위한 치트 또는 저장 데이터 변조

## 4. 책임과 변경 대상

| 대상 | 책임 |
| --- | --- |
| `data/build.json` | 제품명, 표시 버전과 개발 단계의 단일 기준 |
| `GameManager` | 빌드 정보 로드와 안전한 기본값 제공 |
| `GameUI` | 런타임 빌드 라벨 표시 |
| `tools/prepare-web-build.mjs` | Web HTML에 같은 빌드 식별자 삽입 |
| `tools/verify-web-build.mjs` | iPad/PWA 릴리스 산출물 자동 검사 |
| `docs/testing/ipad_v0.2_checklist.md` | 실기기 절차, 판정 기준과 결과 기록 |

## 5. 자동 완료 조건

- 런타임과 Web HTML의 표시 버전이 `data/build.json`과 일치합니다.
- Web manifest가 standalone, landscape와 144/180/512 아이콘을 포함합니다.
- HTML이 `viewport-fit=cover`, 동적 viewport와 고정 canvas 설정을 포함합니다.
- 서비스 워커가 gzip WASM을 캐시하고 즉시 활성화합니다.
- 원본 WASM은 배포 폴더에서 제거되고 정상 gzip 파일만 남습니다.
- Godot import, 전체 headless 테스트, Web export, `prepare:game`,
  `verify:game`, Vinext build와 `git diff --check`가 통과합니다.

## 6. V0.2 최종 완료 조건

자동 완료 조건에 더해 실제 iPad에서 다음 항목이 모두 통과해야 V0.2를 완료로
판정합니다.

1. Safari 가로·세로 화면에서 상·하단 UI와 버튼이 잘리지 않습니다.
2. 터치 부스트와 성장 버튼이 한 번의 터치에 정상 반응합니다.
3. 목도리·운동화·가방·쥐구멍 티어가 배경 위에서 식별됩니다.
4. 지역 발견 카드가 생산을 막지 않고 3초 뒤 사라집니다.
5. Safari 탭 복귀와 홈 화면 PWA 재실행 뒤 저장이 유지됩니다.
6. 화면에 표시된 빌드가 검증 대상 버전과 일치합니다.

실제 기기 결과가 없는 동안 상태는 `실기기 검증 대기`로 유지합니다.
