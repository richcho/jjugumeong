# V0.4.0 작업 계획 — 새끼쥐 보육실

작성일: 2026-07-19
상태: 구현·자동 검증 완료

## 1. 목적

현재 `새끼쥐 보육실`은 쥐구멍 Lv.10 다음 목표 문구일 뿐 실제 시설과 기능이
없습니다. V0.4.0은 보육실을 `건설 → 새끼 돌봄 → 시간 성장 → 성체 합류`가
연결되는 첫 군락 생활 시스템으로 구현합니다.

## 2. 플레이 흐름

```text
쥐구멍 Lv.10 달성
  → 치즈 500으로 보육실 건설
  → 치즈를 사용해 새끼쥐 보육 시작
  → 보육실에서 빛나는 새끼 점을 직접 눌러 돌봄
  → 돌봄 1회마다 성장 시간 15초 단축, 최대 3회
  → 성장 시간이 끝나면 성체 합류
  → mouse_count 증가와 실제 채집조 재구성
```

## 3. 확정 수치

| 항목 | 값 |
| --- | ---: |
| 해금 조건 | 쥐구멍 Lv.10 |
| 건설 비용 | 치즈 500 |
| 보육실 Lv.1 슬롯 | 2 |
| 새끼 등록 비용 | 기본 치즈 200, 완료 수마다 1.5배 |
| 기본 성장 시간 | 120초 |
| 돌봄 시간 단축 | 1회 15초 |
| 새끼당 최대 돌봄 | 3회 |

Alpha 배수는 생산에만 적용하고 시설 비용에는 별도 배수를 적용하지 않습니다.

## 4. 저장 스키마 5

새 영구 필드:

```text
nursery_level
nursery_pups[]
  id
  ready_unix
  care_count
total_raised_pups
next_pup_id
```

- 스키마 4 이하 저장은 미건설 보육실과 빈 새끼 목록으로 이관합니다.
- 새끼의 남은 시간 대신 완료 Unix 시간을 저장해 앱 종료 중에도 성장합니다.
- 완료된 새끼는 자동으로 성체가 되지 않고 사용자가 `성체 합류`를 눌러야
  `mouse_count`가 증가합니다.

## 5. 화면과 직접 참여

- 하단 `보육실` 버튼으로 전용 패널을 엽니다.
- 미건설 상태에서는 해금 조건, 비용과 건설 버튼을 표시합니다.
- 건설 후에는 슬롯, 성장 시간과 새끼별 상태를 표시합니다.
- `NurseryView`는 보육실을 따뜻한 보호 원, 새끼를 작은 점, 돌봄쥐를 순환하는
  점으로 표현합니다.
- 성장 중인 빛나는 새끼 점을 직접 누르면 돌봄이 적용됩니다.
- 색만으로 구분하지 않도록 점 크기, 보호 원과 맥동을 함께 사용합니다.

## 6. 책임

| 구성 | 책임 |
| --- | --- |
| `GameManager` | 비용, 슬롯, 성장 시간, 돌봄, 성체 합류와 저장 상태 |
| `NurseryView` | 점 렌더링, 새끼 좌표 판정과 돌봄 입력 신호 |
| `GameUI` | 패널 구성, 상태 표시와 GameManager 호출 |
| `SaveManager` | 스키마 5 마이그레이션 |

UI와 렌더러는 치즈나 쥐 수를 직접 변경하지 않습니다.

## 7. 제외 범위

- 보육실 Lv.2 이상
- 유전, 희귀도와 부모 선택
- 질병, 부상과 새끼 손실
- 직업 자동 배치
- 시설 내부 전체 지도
- Cloudflare 독립 배포

## 8. 완료 조건

- Lv.10 미만에서는 건설할 수 없고 이유가 표시됩니다.
- 비용이 부족하면 건설과 새끼 등록이 거부됩니다.
- 슬롯을 초과해 새끼를 등록할 수 없습니다.
- 새끼 점을 직접 눌러야 돌봄 시간이 단축됩니다.
- 돌봄은 새끼당 최대 3회만 적용됩니다.
- 완료 전에는 성체 합류가 거부됩니다.
- 완료 후 합류하면 쥐 수가 정확히 1 증가하고 새끼 슬롯이 비워집니다.
- 저장 후 재실행해도 완료 시간과 돌봄 수가 유지됩니다.
- 스키마 1~4 저장이 스키마 5로 이관됩니다.
- 세로·가로 대표 화면에서 보육실 패널이 잘리지 않습니다.
- Godot 전체 테스트, Web export, PWA 검증과 프로덕션 빌드가 통과합니다.

## 9. 구현 결과

- 저장 스키마 5와 스키마 1~4 마이그레이션을 추가했습니다.
- `NurseryView`가 보호 원, 두 슬롯, 새끼 점, 돌봄 표시와 터치 판정을 담당합니다.
- 건설, 등록, 돌봄, 성장 판정과 성체 합류는 `GameManager`에만 구현했습니다.
- 성체 합류 시 `mouse_count_changed`를 발생시켜 실제 채집조를 다시 구성합니다.
- Godot 전체 테스트, 메인 씬 headless 실행, Web export, PWA 검증과 Vinext
  프로덕션 빌드를 통과했습니다.
