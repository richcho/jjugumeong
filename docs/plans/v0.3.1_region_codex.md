# V0.3.1 작업 계획 — 지역 도감과 첫 발견 보상

작성일: 2026-07-19
상태: 구현 및 자동 검증 완료

## 1. 작업 목적

V0.3.0에서 지역 재방문과 사건 완료 기록을 구현했지만, 지역 목록은 이동 기능에
치우쳐 자원·위험·발견 상태를 비교하기 어렵고 첫 사건 해결의 수집 보상도
구분되지 않습니다. 기존 사건 완료 ID를 도감 발견의 단일 기준으로 재사용해
지역 수집 루프를 완성합니다.

## 2. 이번 범위

1. 빌드를 r4 0.3.1로 갱신합니다.
2. 지역 사건 데이터에 첫 발견 추가 보상을 운반 횟수 단위로 정의합니다.
3. 사건 최초 해결 여부를 결과에 포함하고 최초 한 번만 추가 보상을 지급합니다.
4. 지역 패널을 `지역 도감 · 이동`으로 확장합니다.
5. 해금 지역의 자원, 위험, 사건 이름과 발견 상태를 한 화면에 표시합니다.
6. 전체 해금 지역 중 발견한 사건 수를 도감 진행도로 표시합니다.
7. 반복 사건은 기존 기본 보상만 지급하고 첫 발견 보상을 다시 주지 않습니다.
8. 보상, 도감 진행도, 반복 방지와 모바일 모달 경계를 자동 테스트합니다.
9. 사용자 피드백에 따라 쥐의 등·꼬리 장비 도형, 쥐구멍 주변 장식선과
   쥐 추가 시 굴 앞에 따로 표시되던 장식쥐를 제거합니다.

## 3. 상태와 저장

- 발견 기준은 기존 `completed_region_event_ids`입니다.
- 첫 발견 보상 지급 여부도 같은 ID로 판정합니다.
- 새 영구 필드가 없으므로 저장 스키마 2를 유지합니다.
- V0.3.0에서 이미 완료한 사건은 발견 완료로 간주하며 추가 보상을 소급하지
  않습니다.

## 4. 첫 발견 보상

각 지역의 첫 사건 해결은 선택지 기본 보상에 운반 3회분을 추가합니다. 보상은
현재 지역 생산 보너스와 Alpha 배수를 그대로 사용합니다. 반복 해결은 선택지에
정의된 1회 또는 2회 운반 보상만 지급합니다.

## 5. 제외 범위

- 잠긴 지역 정보의 미리 공개
- 다수 사건을 가진 완전한 지역별 도감
- 별도 도감 화폐와 수집 상점
- 업적, 퀘스트와 알림 배지
- 저장 스키마 변경

## 6. 완료 조건

- 모든 지역 사건에 첫 발견 보상이 명시됩니다.
- 최초 해결 결과와 반복 해결 결과의 보상이 자동 테스트로 구분됩니다.
- 지역 도감에 자원·위험·사건·발견 상태와 진행도가 표시됩니다.
- 이미 완료한 사건은 재실행 후에도 첫 보상을 다시 받지 않습니다.
- Godot 전체 테스트, Web export, PWA 검증과 프로덕션 빌드가 통과합니다.

## 7. 구현 결과

- 5개 지역 사건에 첫 발견 운반 3회분 보상을 정의했습니다.
- 최초 발견은 부엌 기본 상태 기준 치즈 125, 같은 안전 선택의 반복은 치즈
  50으로 자동 검증했습니다.
- 도감은 해금 지역만 대상으로 자원·위험·사건·발견 여부와 발견 진행도를
  표시합니다.
- 새 영구 필드를 추가하지 않고 기존 완료 사건 ID를 사용해 스키마 2를
  유지했습니다.
- 실제 왕복 쥐와 치즈 운반 스프라이트는 유지하고, 배경·스프라이트 위에
  덧그린 절차적 장비·굴 장식·굴 앞 장식쥐만 제거했습니다.
- Godot 4.7 import, 전체 테스트, 메인 씬 headless 실행, Web release export,
  Web 검증, Vinext 프로덕션 빌드와 `git diff --check`를 통과했습니다.
