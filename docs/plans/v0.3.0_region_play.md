# V0.3.0 작업 계획 — 지역 재방문과 첫 선택 사건

작성일: 2026-07-19
상태: 구현·자동 검증·프로덕션 배포 완료 · iPad 확인 대기

## 1. 선행 상태

사용자가 r4 0.2.7 iPad 빌드의 정상 동작을 확인했습니다. V0.2는 완료로
전환하고 V0.3 지역 플레이의 첫 수직 슬라이스를 시작합니다.

## 2. 작업 목적

현재 `current_stage_index`는 최고 해금 지역과 현재 화면 지역을 동시에 뜻합니다.
이 상태에서는 이전 지역 재방문이 다음 지역 진행도와 저장을 깨뜨립니다. 최고
해금 상태와 선택 지역을 분리한 뒤, 재방문할 이유가 되는 지역별 2선택 사건을
연결합니다.

## 3. 이번 범위

1. 최고 해금 지역과 현재 방문 지역을 별도 런타임 상태로 관리합니다.
2. 해금 지역 ID 목록, 선택 지역과 발견 사건 ID를 저장합니다.
3. 저장 스키마를 1에서 2로 올리고 기존 저장을 자동 마이그레이션합니다.
4. 해금된 지역만 표시하는 지역 선택 UI와 즉시 배경 전환을 구현합니다.
5. 각 지역에 첫 선택 사건 1개와 두 개의 대응 선택지를 데이터로 정의합니다.
6. 선택 결과는 무작위 손실 없이 `안전한 추가 보상` 또는 `짧은 이동 가속`으로
   구분합니다.
7. 사건 완료 기록과 재실행 악용을 막는 사건 쿨다운을 저장합니다.
8. 선택·마이그레이션·사건 결과·기존 저장 호환을 자동 테스트합니다.

## 4. 제외 범위

- 자유 이동 지도와 지역 간 이동 시간
- 전투, 체력, 장비 소모와 영구 손실
- 다단계 대화와 분기 서사
- 쥐 직업, 희귀도와 배치
- 서버 및 온라인 상태

## 5. 상태 모델

| 상태 | 책임 | 저장 |
| --- | --- | --- |
| `highest_unlocked_stage_index` | 누적 치즈로 도달한 최고 지역 | ID 목록으로 복원 |
| `current_stage_index` | 현재 방문하고 생산하는 지역 | `selected_stage_index` |
| `unlocked_stage_ids` | 해금 지역의 안정적인 식별자 | 저장 |
| `completed_region_event_ids` | 한 번 이상 해결한 사건 도감 | 저장 |
| `next_region_event_unix` | 재실행 후에도 유지되는 사건 쿨다운 | 저장 |

해금 권한의 최종 기준은 누적 치즈와 스테이지 데이터입니다. 저장된 ID가
누락되거나 잘못돼도 로드 시 안전하게 재구성합니다.

## 6. 저장 스키마 2

추가 필드:

- `selected_stage_index`
- `unlocked_stage_ids`
- `completed_region_event_ids`
- `next_region_event_unix`

스키마 1 저장은 기존 `current_stage_index`를 선택 지역으로 사용하고, 누적 치즈로
해금 ID를 재구성합니다. 기존 자원·강화·통계 필드는 변경하지 않습니다.

## 7. 완료 조건

- 이전 지역을 선택해도 최고 해금 지역과 다음 보상 목표가 뒤로 가지 않습니다.
- 잠긴 지역은 선택할 수 없습니다.
- 선택 지역이 저장·재실행 뒤 유지됩니다.
- 스키마 1 저장이 손실 없이 스키마 2로 변환됩니다.
- 각 지역 사건에 정확히 두 선택지가 있고 결과가 게임 상태 계층을 거칩니다.
- 사건 해결 기록과 쿨다운이 저장됩니다.
- Godot import, headless 테스트, Web export, PWA 검증과 프로덕션 빌드가
  통과합니다.

## 8. 구현 결과

- 최고 해금 지역과 현재 방문 지역을 분리했습니다.
- 해금 지역 목록에서 이전 배경과 생산 보너스로 즉시 이동할 수 있습니다.
- 다음 지역 보상은 방문 위치가 아니라 최고 해금 위치를 기준으로 유지됩니다.
- 5개 지역에 고유 사건과 정확히 두 선택지를 추가했습니다.
- 사건 발견 ID와 공통 30초 쿨다운을 저장합니다.
- 저장 스키마 2와 스키마 1 마이그레이션을 구현했습니다.
- 지역 목록과 사건 선택 모달을 세로·가로 반응형 UI에 연결했습니다.
