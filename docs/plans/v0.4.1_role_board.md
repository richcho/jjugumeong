# V0.4.1 작업 계획 — 군락 역할 보드

작성일: 2026-07-19
상태: 구현·자동 검증 완료

## 1. 목적

보육실에서 늘어난 쥐가 모두 같은 채집 행동만 반복하는 한계를 해결합니다.
V0.4.1은 전체 인구를 `채집·탐험·건설` 역할에 직접 배치하고, 선택이 생산,
지역 활동과 다음 새끼 성장에 실제 차이를 만드는 첫 직업 수직 슬라이스입니다.

## 2. 사용자 흐름

```text
쥐 3마리 이상 확보
  → 하단 역할 보드 열기
  → 채집·탐험·건설 역할점을 직접 누르기
  → 채집쥐 한 마리가 선택한 전문 역할로 이동
  → 채집점 누르기로 전문쥐를 한 마리씩 복귀
  → 생산·지역 보상·새끼 성장 시간 변화 확인
```

`전체 채집 복귀` 버튼으로 언제든 기본 배치로 되돌릴 수 있습니다.

## 3. 확정 규칙

| 역할 | 효과 |
| --- | --- |
| 채집쥐 | 실제 왕복 채집조와 초당 생산에 참여 |
| 탐험쥐 | 지역 사건·현장 행동 보상 1마리당 10% 증가, 최대 30% |
| 건설쥐 | 새로 등록하는 새끼의 기본 성장 시간 1마리당 10% 단축, 최대 30% |

- 군락에는 항상 채집쥐가 최소 1마리 남습니다.
- 전문 역할점을 누르면 채집쥐 한 마리가 이동합니다.
- 채집 역할점을 누르면 건설쥐, 탐험쥐 순서로 한 마리가 복귀합니다.
- 역할을 바꾸어도 이미 시작한 새끼의 완료 시각은 바뀌지 않습니다.
- 새로 합류하거나 구매한 쥐는 기본적으로 채집 역할에 들어갑니다.

## 4. 점 기반 표현

- `RoleBoardView`는 세 역할을 크기와 색이 다른 세 개의 큰 역할점으로 표시합니다.
- 배치된 쥐는 역할점 주위를 도는 작은 점으로 표시합니다.
- 역할점의 이름, 현재 수, 실제 효과를 문자로 함께 표시해 색에만 의존하지 않습니다.
- 마우스와 터치는 같은 좌표 판정 함수를 사용합니다.

## 5. 저장 스키마 6

새 필드:

```text
role_assignments
  gatherer
  explorer
  builder
```

- 스키마 5 이하 저장은 모든 기존 쥐를 채집 역할로 이관합니다.
- 로드 시 합계가 전체 쥐 수와 다르면 전문 역할을 먼저 보존하되 최소 채집쥐
  1마리를 확보하고 나머지를 채집 역할에 배치합니다.

## 6. 책임

| 구성 | 책임 |
| --- | --- |
| `GameManager` | 역할 합계, 배치 규칙, 효과 계산과 저장 |
| `RoleBoardView` | 점 렌더링, 역할점 좌표 판정과 입력 신호 |
| `GameUI` | 상태 설명, 역할 보드 열기와 GameManager 호출 |
| `WorldView` | 전체 인구가 아닌 현재 채집쥐 수로 채집조 구성 |
| `SaveManager` | 스키마 6 마이그레이션 |

## 7. 제외 범위

- 개별 쥐 이름·희귀도·특성
- 부모 선택과 유전
- 직업 레벨과 장비
- 자동 최적 배치
- 건설 자원과 건물 공사
- Cloudflare 독립 배포와 사용자 도메인 변경

## 8. 완료 조건

- 쥐 3마리 미만에서는 역할 변경이 잠기고 이유가 표시됩니다.
- 역할 배치 합계는 항상 `mouse_count`와 같습니다.
- 채집쥐는 최소 1마리이며 실제 채집조와 예상 생산이 함께 줄고 늘어납니다.
- 탐험쥐 보너스는 지역 사건과 현장 행동에만 적용됩니다.
- 건설쥐 보너스는 역할 변경 뒤 새로 시작한 새끼 성장 시간에만 적용됩니다.
- 새 쥐 합류 시 채집 역할이 정확히 1 증가합니다.
- 저장 후 재실행해도 역할 배치가 유지됩니다.
- 스키마 1~5 저장이 스키마 6으로 이관됩니다.
- 세로·가로 대표 화면에서 역할 보드가 잘리지 않습니다.
- Godot 전체 테스트, Web export, PWA 검증과 프로덕션 빌드가 통과합니다.

## 9. 구현 결과

- 역할 합계와 효과 계산을 `GameManager`에 구현하고 저장 스키마 6으로
  영구화했습니다.
- `RoleBoardView`가 세 역할점, 배치 점의 궤도와 마우스·터치 좌표 판정을
  담당합니다.
- `WorldView`는 채집쥐 수로만 실제 채집조를 다시 구성합니다.
- 탐험 보너스는 지역 사건·현장 행동에만, 건설 보너스는 새로 시작하는 새끼의
  완료 시각에만 적용됩니다.
- 빈 저장과 저장 재실행에서 Godot 전체 테스트를 연속 통과했습니다.
- 메인 씬 headless 실행, Web export, PWA 검사와 Vinext 프로덕션 빌드를
  통과했습니다.
- 연결된 소유자 전용 Sites 프로덕션 배포를 완료했습니다.
