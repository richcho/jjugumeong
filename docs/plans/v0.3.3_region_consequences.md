# V0.3.3 작업 계획 — 지역 행동과 지속 결과

작성일: 2026-07-19
상태: 구현 및 자동 검증 완료

## 1. 문제

V0.3.2에서 사용자가 점을 직접 누르는 세 가지 현장 행동을 추가했지만, 완료
결과는 치즈 보상과 완료 표시로 끝납니다. 행동이 지역의 위험, 이동 경로와 다음
사건을 바꾸지 않으므로 반복 플레이의 의미가 약합니다.

V0.3.3의 목표는 `직접 행동 → 지역 변화 → 달라진 사건 선택 → 다음 행동`이
이어지는 지역 참여 루프를 완성하는 것입니다.

## 2. 완료할 지역 행동

| 지역 | 직접 행동 | 조작 규칙 | 첫 지속 결과 |
| --- | --- | --- | --- |
| 낡은 부엌 | 실타래 속 비밀길 | 연결된 매듭 순서 추적 | 실 통로 개방 |
| 식료품 창고 | 선반 철탑 오르기 | 아래부터 교차 발판 상승 | 높은 우회로 확보 |
| 동네 편의점 | 무한 진열 통로 | 중앙 신호와 같은 문 선택 | 진열 통로 지도 완성 |
| 대형 식당 | 화로 사이 통과 | 열기 파동의 안전 구간에 맞춰 이동 | 서늘한 배수관 개방 |
| 치즈 공장 | 컨베이어 전환 | 분기점 방향을 바꿔 치즈점 유도 | 안전 운반선 제어 |

다섯 행동은 배경과 문장만 다른 동일 버튼 구조로 만들지 않습니다. 식당은 시간
판정, 공장은 경로 연결을 사용해 기존 세 행동과 조작을 구분합니다.

## 3. 지역별 영구 상태

각 해금 지역은 다음 상태를 저장합니다.

```text
region_progress[stage_id]
  action_level       현장 행동 숙련도 0~3
  risk_level         지역 위험도 0~3
  route_unlocked     안전 경로 개방 여부
  flags              지역별 발견·해결 표식
  last_choice_id     마지막 사건 선택
```

- 첫 행동 완료는 `route_unlocked`를 열고 지역 고유 표식을 추가합니다.
- 반복 완료는 `action_level`을 최대 3까지 올립니다.
- 행동 성공은 위험도를 낮추고, 무시하거나 위험한 사건 선택은 위험도를 높일 수
  있습니다.
- 지역 상태는 화면 표시, 생산 보너스와 사건 선택 조건의 단일 기준이 됩니다.

새 영구 상태가 추가되므로 저장 스키마를 3에서 4로 올립니다. 스키마 3 이하
저장은 해금 지역마다 기본 상태를 생성해 이관합니다.

## 4. 실제 게임 효과

| 상태 | 효과 |
| --- | --- |
| 안전 경로 개방 | 해당 지역 생산량 5% 증가 |
| 행동 숙련도 2 | 현장 행동 기본 보상 1회분 증가 |
| 행동 숙련도 3 | 사건의 지역 전문 선택 해금 |
| 위험도 0 | 안전 상태, 사건 대기 시간 20% 감소 |
| 위험도 3 | 위험 상태, 고보상 선택과 복구 행동 동시 표시 |

효과 수치는 `GameManager`에서 계산하고 UI나 점 렌더러가 직접 자원을 변경하지
않습니다.

## 5. 지역 사건 재설계

기존의 고정된 `안전 / 질주` 두 선택지를 동적 2~3개 선택으로 바꿉니다.

각 선택 데이터는 다음을 포함합니다.

```text
id
label
description
requirements       필요한 경로, 숙련도, 치즈 또는 쥐 수
cost               선택 즉시 지불할 자원
reward_trips
risk_delta
add_flags
remove_flags
result
```

UI는 선택 전에 비용, 조건과 예상 변화를 보여 줍니다. 조건이 부족한 선택은
숨기지 않고 잠금 이유를 표시합니다.

## 6. 지역별 사건 연결

### 낡은 부엌

- 실 통로가 열리면 고양이 순찰을 피하는 전문 선택 해금
- 미끼 선택은 치즈를 소비하지만 위험도를 크게 낮춤
- 무리한 질주는 보상이 크지만 위험도가 올라감

### 식료품 창고

- 높은 우회로가 열리면 쥐덫 구간을 건너지 않음
- 철탑 숙련도가 높으면 쥐덫 해체 선택 해금

### 동네 편의점

- 통로 지도를 완성하면 인간의 청소 동선을 미리 표시
- 잘못된 시간에 진입하면 인간 주의도에 해당하는 위험도가 상승

### 대형 식당

- 화로 안전 구간을 통과하면 배수관 경로 개방
- 위험도가 높으면 화재 진압 또는 고급 식량 선점 중 하나를 선택

### 치즈 공장

- 컨베이어 제어 후 자동 운반 선택 해금
- 기계 위험도가 높으면 생산 보너스보다 안전 복구를 먼저 선택할 수 있음

## 7. 결과 표현

행동과 사건 완료 뒤 결과 패널에 다음 네 가지를 반드시 표시합니다.

1. 사용자가 수행한 행동
2. 획득하거나 지불한 자원
3. 바뀐 지역 상태
4. 새로 열린 다음 선택

지역 도감에는 행동 숙련도, 위험도, 안전 경로와 마지막 선택을 표시합니다.
`DotActionView`에는 현재 위험도와 열린 경로를 점의 색, 밀도와 연결선으로
표현합니다. 색만으로 상태를 판정하지 않도록 크기와 선 형태를 함께 사용합니다.

## 8. 구현 순서

1. `region_progress` 데이터 모델과 스키마 4 마이그레이션
2. 공통 지역 효과 계산과 저장 왕복 테스트
3. 식당 시간 판정 행동 구현
4. 공장 경로 전환 행동 구현
5. 기존 세 행동의 지속 결과 연결
6. 동적 사건 선택·조건·비용 UI 구현
7. 결과 패널과 지역 도감 상태 표시
8. 전체 Godot·Web·iPad 레이아웃 검증
9. 문서·변경 기록·배포

## 9. 제외 범위

- 보육실과 새끼 성장
- 쥐 직업과 영웅
- 파주·문산 또는 실제 군사 경계 지역
- Cloudflare 독립 배포
- 전투, 부상과 영구 개체 손실
- 온라인 기능

보육실은 V0.3.3 완료 뒤 `보육 → 성장 → 성체 합류`를 독립 수직 슬라이스로
설계합니다.

## 10. 완료 조건

- 다섯 지역에 서로 다른 직접 조작이 존재합니다.
- 모든 첫 행동 완료가 해당 지역 상태를 실제로 변경합니다.
- 열린 경로가 생산량과 사건 선택에 함께 영향을 줍니다.
- 조건이 다른 사건 선택이 최소 하나씩 존재하고 잠금 이유가 표시됩니다.
- 사건 결과가 위험도, 표식과 마지막 선택으로 저장됩니다.
- 스키마 1~3 저장이 스키마 4로 안전하게 이관됩니다.
- 저장 후 재실행해도 지역 상태와 선택 조건이 동일합니다.
- 글자만 다르고 비용·조건·결과가 같은 선택은 완료로 인정하지 않습니다.
- 768×1024와 1366×1024에서 행동·사건·결과 패널이 잘리지 않습니다.
- Godot 전체 테스트, Web export, PWA 검증과 프로덕션 빌드가 통과합니다.
- 실제 iPad에서 다섯 행동의 터치 판정을 사용자가 확인해야 최종 완료로
  전환합니다.

## 11. 구현 결과

- 다섯 지역에 실타래·철탑·색 문·열기 타이밍·컨베이어 경로 행동을 연결했습니다.
- 첫 완료는 안전 경로를 열고 위험도를 낮추며, 반복 완료는 숙련도를 3까지
  올립니다.
- 안전 경로는 생산량 5%와 지역 사건 보상에 적용됩니다.
- 숙련 Lv.3에서 지역별 전문 선택이 열리고 이전에는 잠금 이유를 표시합니다.
- 안전·질주·전문 선택이 위험도와 마지막 선택을 변경해 스키마 4로 저장됩니다.
- 스키마 1~3 마이그레이션, 다섯 입력 방식, 전문 선택 해금, 세로 UI와 Web
  빌드를 자동 검증했습니다.
