# 스코프 재정렬 작업 계획 — 1일 완결 목표 정합화

작성일: 2026-07-20
상태: 계획 (실행 대기)
관련 문서: `jjugumeong_master_plan.md`, `game_design.md`, `roadmap.md`, `balancing.md`, `world_lore.md`

이 문서는 새 기능 추가 계획이 아니라, 기존 범위를 "1일 안에 완전 정복
가능한 방치형 게임"이라는 목표에 맞춰 재정렬하는 선행 작업 계획입니다.
V0.5 착수 전에 이 작업을 먼저 완료합니다.

이 문서는 원래 "6시간 완전 정복"을 목표로 작성됐던 `scope_review_6h_target.md`를
대체합니다(2026-07-20, git mv로 이름 변경 및 재작성). 6시간이라는 숫자는
실행 전 상태에서 사용자 결정으로 1일(연속 세션 기준 24시간)로 교체됐습니다.
같은 날 추가로, "1일 정도면 충분하고 정밀한 실측 플레이까지는 필요 없다"는
결정에 따라 이 문서의 방법론도 실측 대신 설계 단계 추정으로 전환했습니다.

## 1. 목적

현재 게임은 매 버전(V0.1~V0.4.5)마다 새 시스템이 추가되며 성장했지만,
총 플레이 시간이 목표와 대략적으로도 맞는지 가늠된 적이 없습니다. 동시에
기준 문서(로드맵, 세계관)는 은하 시대까지 이어지는 장기 비전을 정본으로
유지하고 있어, 매 버전 기획 시 "이게 목표 범위의 게임에 필요한가"라는
질문이 구조적으로 걸러지지 않습니다.

**목표 정의**: 목표 시간은 **24시간(1일) 내외**입니다. 정밀한 실측
플레이(처음부터 끝까지 실제로 완주하며 시계로 재는 것)는 요구하지
않으며, 설계 단계의 추정으로 충분합니다. 추정은 `balancing.md` §1의
예상 생산 공식(왕복 시간·초당 생산), §3의 중간 이정표 후보, 그리고 이미
수치가 확정된 메타시스템(보육실 성장 120초, 영웅 임무 재도전 60초 등)을
근거로 계산합니다. 이 24시간이라는 목표 자체도 정밀 검증된 값이 아니라
사용자가 내린 대략적인 목표값이며, 이 문서는 그 이상의 정밀도를
요구하지 않습니다.

## 3. 실행 항목

### 3.1 설계 페이스 추정 (최우선, 다른 모든 판단의 입력값)

근거 없는 컷/유지 결정은 피해야 하지만, 그 근거가 반드시 실제 완주
측정일 필요는 없습니다.

- 방법: 실제 플레이 측정 대신 다음 근거로 각 막의 예상 소요 시간을
  추정합니다.
  - `balancing.md` §1 예상 생산 공식(왕복 시간, 초당 생산)으로 지역별
    해금까지 걸리는 이론적 시간 계산
  - `balancing.md` §3 중간 이정표 후보 표의 30%/60% 지점을 지역 구간
    페이스 기준으로 사용
  - 보육실 성장(120초)·역할보드·영웅 유대 임무(60초 재도전)처럼 이미
    수치가 정해진 메타시스템은 그 수치를 그대로 시간 추정에 반영
  - 필요하면 짧은 부분 플레이(5~10분 단위 구간 확인)로 추정치를
    보정할 수 있으나, 처음부터 끝까지 완주하는 정밀 실측은 요구하지
    않습니다.
- 기록 항목 (`balancing.md` 시간예산 원장을 그대로 사용):
  - 첫 강화까지 걸리는 예상 시간
  - 지역별 해금까지 걸리는 예상 시간 (5개 지역 개별)
  - 보육실 첫 건설 ~ 첫 성체 합류까지 예상 시간
  - 역할보드 첫 배치까지 예상 시간
  - 영웅 해금 ~ 유대 Lv.3 도달까지 예상 시간
  - 총 예상 누적 시간
- 산출물: `balancing.md`의 시간예산 원장에 막별 추정치와 근거를 채우고,
  결정 대기 항목 "중간 이정표 정확한 수치"를 추정 근거를 명시한 채로
  확정으로 전환합니다.
- 판단 기준: 추정 총 시간이 24시간(±2~3시간) 범위를 크게 벗어나면 3.4의
  컷 후보를 우선순위대로 적용합니다.

### 3.2 문서 스코프 경계선 명시

- `jjugumeong_master_plan.md`에 "1일 스코프 경계"(§10) 섹션을 신설합니다.
  - IN: V0.1~V0.4 구현 전체 + V0.5~V1.0("첫 정식 문명")
  - OUT (부록으로 격리): `roadmap.md` 장기 시대 6종 전부, `world_lore.md`
    문명 단계 중 "지하 국가 연합" 이후 전부, 미구현 영웅 5명·세력 6개
  - **예산이 6시간→24시간(최대 4배)으로 늘었다고 해서 장기 시대가 스코프로
    복귀하는 것은 아닙니다.** 확정 스코프 경계는 여전히 V1.0까지입니다.
- 이관 방법: 로드맵 본문에서 삭제하지 않고 "부록: 장기 시대 메모 (스코프 밖,
  참고용)"으로 이동하며, 부록 최상단에 "이 부록은 우선순위 판단에 사용하지
  않음"을 명시합니다.

### 3.3 미니게임 통합 리팩터링

- 대상: `dot_action_view.gd`, `hero_bond_view.gd`, 지역 사건 2택1 로직
- 방향: 점 개수, 순서 규칙, 실수 시 후퇴 여부, 보상 콜백만 파라미터로
  받는 공통 컴포넌트(가칭 `SequenceActionView`)로 통합
- 이 통합은 `jjugumeong_master_plan.md` §12 "연계 원칙"의 직접적인 실행
  근거입니다 — 조작 방식이 시스템마다 달라 단절되어 보이는 문제를 해소합니다.
- 선행 조건: 3.1 추정 이후 착수 (추정 대상 흐름을 리팩터링 중간에
  건드리지 않기 위함)
- 완료 조건: 기존 3개 기능(부엌 실타래·창고 철탑·편의점 통로, 영웅 유대
  임무)이 통합 컴포넌트 위에서 동일하게 동작하고 저장 스키마는 변경하지
  않습니다.

### 3.4 메타시스템 예산 정합 판단

대상: 보육실, 역할보드, 영웅, 영웅유대 4종

설계 추정 기반 판단 기준(그랜드토탈 임계값이 아니라 **소속 막 예산 대비**로
판단):

- 소속 막(2막 무리)의 추정 시간이 목표 예산(1.5–2.5h)을 초과하면: 역할보드를
  1순위 컷/병합 후보로 검토 (탐험쥐·건설쥐 보너스를 영웅 효과로 흡수하는 방식)
- 여전히 초과하면: 보육실의 점 돌봄 미니게임을 제거하고 단순 대기시간
  기반 자동 성장으로 단순화
- 총 시간이 24시간에 크게 못 미치면: 컷은 불필요하며, 대신 5개 지역
  각각의 밀도를 높이는 방향으로 재배분
- 허용오차는 24시간 규모에 맞춰 ±2~3시간으로 잡습니다. 정밀 실측이
  아닌 추정이므로 오차 범위를 넉넉히 둡니다.

### 3.5 아트 방향 결정

- 결정 대기 보드에 신규 항목 "리얼리즘 vs 스타일화"를 추가합니다.
- 옵션 A: 걷기·운반·정지 3~5프레임 스프라이트를 최소로 확보해 절차적
  모션과 결합
- 옵션 B: 리얼리즘을 포기하고 도형 기반 스타일을 정식 아트 방향으로
  확정, `game_design.md` 4절의 "실물처럼" 표현을 스타일 목표로 수정
- 1일 규모 게임과 개발 리소스를 고려하면 옵션 B를 기본값으로 권장하되
  최종 결정은 사용자가 내립니다.
- 이 항목은 시간예산 재정렬과 무관하게 그대로 유지됩니다.

### 3.6 향후 버전 계획 체크리스트 도입

`jjugumeong_master_plan.md` §11 "시간예산 관리 원칙"에 정의된 6항목 체크리스트를
**V0.5부터** 모든 `docs/plans/vX.Y.Z_*.md` 문서에 적용합니다. V0.1–V0.4.5
기존 계획 문서는 소급 적용하지 않습니다.

### 3.7 게임 경험 정합성 원칙 도입

`jjugumeong_master_plan.md` §12에 5개 원칙(연계·개체 독립성·의도된 재미
정렬·능동 참여 비중·이동 목표지점 정확도)을 등록하고, 추적이 필요한 항목을
결정 대기 보드(§13)에 반영합니다. 이동 정확도는 실제 버그 여부 확인 후
별도 처리(§12.2 분기 참고).

## 6. 제외 범위

- 이 작업은 신규 콘텐츠를 추가하지 않습니다.
- 로드맵 항목의 완전 삭제는 하지 않습니다 (부록 이관만 수행합니다).
- 장기 시대(6막) 콘텐츠는 예산 증가와 무관하게 스코프 밖으로 유지합니다.
- V0.5 지하마을 착수는 이 작업 완료 후로 미룹니다.
- **정밀한 실측 플레이(처음부터 끝까지 실제로 24시간 완주하며 측정하는
  것)는 이 계획에서 요구하지 않습니다.** 설계 단계 추정으로 충분합니다.
- 코드 변경은 이 문서 작업의 범위 밖입니다.
