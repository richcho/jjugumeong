# 스코프 재정렬 작업 계획 — 6시간 완결 목표 정합화

작성일: 2026-07-20
상태: 계획 (실행 대기)
관련 문서: `jjugumeong_master_plan.md`, `game_design.md`, `roadmap.md`, `balancing.md`, `world_lore.md`

이 문서는 새 기능 추가 계획이 아니라, 기존 범위를 "6시간 안에 완전 정복
가능한 방치형 게임"이라는 목표에 맞춰 재정렬하는 선행 작업 계획입니다.
V0.5 착수 전에 이 작업을 먼저 완료합니다.

## 1. 목적

현재 게임은 매 버전(V0.1~V0.4.5)마다 새 시스템이 추가되며 성장했지만,
총 플레이 시간이 6시간 목표와 맞는지 한 번도 측정되지 않았습니다. 동시에
기준 문서(로드맵, 세계관)는 은하 시대까지 이어지는 장기 비전을 정본으로
유지하고 있어, 매 버전 기획 시 "이게 6시간짜리 게임에 필요한가"라는
질문이 구조적으로 걸러지지 않습니다.

## 3. 실행 항목

### 3.1 실측 (최우선, 다른 모든 판단의 입력값)

측정 없이는 어떤 컷/유지 결정도 근거가 없습니다.

- 방법: Alpha 배수(25배)를 켠 상태 1회, 끈 상태(정식 밸런스 근사치) 1회,
  각각 처음부터 5개 지역 + 보육실 + 역할보드 + 영웅 + 영웅유대 Lv.3까지
  완주하며 시간을 기록합니다.
- 기록 항목 (`balancing.md` 4절 관찰 지표를 그대로 사용):
  - 첫 강화까지 걸린 시간
  - 지역별 해금까지 걸린 시간 (5개 지역 개별 기록)
  - 보육실 첫 건설 ~ 첫 성체 합류까지 걸린 시간
  - 역할보드 첫 배치까지 걸린 시간
  - 영웅 해금 ~ 유대 Lv.3 도달까지 걸린 시간
  - 총 누적 시간
- 산출물: `balancing.md`에 실측 표를 추가하고, 결정 대기 항목 "중간
  이정표 정확한 수치"를 확정으로 전환합니다.
- 판단 기준: 총 시간이 6시간(±1시간) 범위를 벗어나면 3.4의 컷 후보를
  우선순위대로 적용합니다.

### 3.2 문서 스코프 경계선 명시

- `jjugumeong_master_plan.md`에 "6시간 스코프 경계" 섹션을 신설합니다.
  - IN: V0.1~V0.4 구현 전체 + V0.5 지하마을 중 확정된 항목만
  - OUT (부록으로 격리): `roadmap.md` V0.6 이후 전부, `world_lore.md`
    문명 단계 중 "지하 국가 연합" 이후 전부, 미구현 영웅 5명·세력 6개
- 이관 방법: 로드맵 본문에서 삭제하지 않고 "부록: 장기 메모 (스코프 밖,
  참고용)"로 이동하며, 부록 최상단에 "이 부록은 우선순위 판단에 사용하지
  않음"을 명시합니다.

### 3.3 미니게임 통합 리팩터링

- 대상: `dot_action_view.gd`, `hero_bond_view.gd`, 지역 사건 2택1 로직
- 방향: 점 개수, 순서 규칙, 실수 시 후퇴 여부, 보상 콜백만 파라미터로
  받는 공통 컴포넌트(가칭 `SequenceActionView`)로 통합
- 선행 조건: 3.1 실측 이후 착수 (실측 대상 흐름을 리팩터링 중간에
  건드리지 않기 위함)
- 완료 조건: 기존 3개 기능(부엌 실타래·창고 철탑·편의점 통로, 영웅 유대
  임무)이 통합 컴포넌트 위에서 동일하게 동작하고 저장 스키마는 변경하지
  않습니다.

### 3.4 메타시스템 축소 판단

대상: 보육실, 역할보드, 영웅, 영웅유대 4종

실측 데이터 기반 판단 기준:

- 총 시간이 6시간을 초과하면: 역할보드를 1순위 컷/병합 후보로 검토
  (탐험쥐·건설쥐 보너스를 영웅 효과로 흡수하는 방식)
- 여전히 초과하면: 보육실의 점 돌봄 미니게임을 제거하고 단순 대기시간
  기반 자동 성장으로 단순화
- 총 시간이 6시간에 크게 못 미치면: 컷은 불필요하며, 대신 5개 지역
  각각의 밀도를 높이는 방향으로 재배분

### 3.5 아트 방향 결정

- 결정 대기 보드에 신규 항목 "리얼리즘 vs 스타일화"를 추가합니다.
- 옵션 A: 걷기·운반·정지 3~5프레임 스프라이트를 최소로 확보해 절차적
  모션과 결합
- 옵션 B: 리얼리즘을 포기하고 도형 기반 스타일을 정식 아트 방향으로
  확정, `game_design.md` 4절의 "실물처럼" 표현을 스타일 목표로 수정
- 6시간 게임 규모와 개발 리소스를 고려하면 옵션 B를 기본값으로 권장하되
  최종 결정은 사용자가 내립니다.

## 4. 실행 순서와 의존관계

1. 3.1 실측 — 다른 모든 판단의 입력값
2. 3.5 아트 방향 결정 — 실측과 독립적으로 병행 가능
3. 3.2 문서 스코프 경계선 — 3.1 결과 반영
4. 3.4 메타시스템 축소 판단 — 3.1 결과로 확정
5. 3.3 미니게임 통합 리팩터링 — 3.4 확정 이후, 남길 시스템 범위가
   정해진 뒤 착수

## 5. 완료 조건

- `balancing.md`에 실측 시간표가 채워지고 해당 결정 대기 항목이
  확정으로 전환됨
- `jjugumeong_master_plan.md`에 6시간 스코프 IN/OUT 경계가 명시됨
- `roadmap.md` 장기 항목이 부록으로 분리되어 우선순위 판단에서 제외됨
- 아트 방향이 결정되고 `game_design.md`에 반영됨
- 실측 결과에 따라 메타시스템 4종 중 컷·축소 대상이 확정됨
- (후속) 통합 미니게임 컴포넌트가 기존 저장 스키마를 깨지 않고 동작함
