# V0.4.5 작업 계획 — 살아있는 쥐 움직임

작성일: 2026-07-19
상태: 구현·검증·운영 배포 완료

## 1. 목적

현재 쥐는 경로를 일정한 속도로 왕복하고 도착점에서만 짧게 멈춥니다. 실제
쥐처럼 느껴지려면 그림 교체보다 먼저 이동의 리듬, 탐색, 방향 전환과 운반
자세가 달라져야 합니다. V0.4.5는 기존 자동 생산량과 왕복 책임을 유지하면서
화면의 주인공을 쥐 행동으로 바꿉니다.

## 2. 행동 상태

```text
굴에서 나오기
  → 가속하며 이동
  → 경로 중간의 짧은 킁킁 탐색
  → 다시 가속
  → 자원 앞 감속·확인·치즈 물기
  → 몸을 돌려 무게를 싣고 귀환
  → 굴 앞 감속·보상 입고
  → 굴 안 휴식
```

| 상태 | 시간·조건 | 표현 |
| --- | --- | --- |
| 이동 | 경로 진행 중 | 가속·감속, 불규칙한 보폭과 낮은 몸 |
| 킁킁 탐색 | 출발 후 경로 중간, 0.45~0.70초 | 정지, 코 방향의 짧은 상하 움직임 |
| 자원 확인 | 자원 도착, 0.65~1.19초 | 치즈 확인 뒤 운반 자세 전환 |
| 굴 진입 | 보상 입고 뒤 0.42초 | 속도를 낮추고 작아지며 진입 |
| 휴식 | 2.4~5.65초 | 화면 밖 굴 내부 휴식 |
| 굴 이탈 | 0.48초 | 몸을 낮춘 상태에서 점차 나타남 |

- 킁킁 탐색은 자원에 가까운 마지막 구간과 굴 바로 앞에서는 발생하지 않습니다.
- 각 쥐는 속도, 탐색 간격과 보폭 위상이 달라 동시에 같은 동작을 하지 않습니다.
- 운반 중에는 보폭과 몸 튀김을 줄이고 가속을 낮춥니다.
- 탐색 정지와 운반 감속이 기존 평균 왕복 시간을 과도하게 늘리지 않도록 실제
  이동 구간 속도에 1.12배 보정값을 적용합니다.

## 3. 시각 운동

- 실제 이동속도를 기준으로 보폭 위상과 몸의 운동량을 계산합니다.
- 출발할 때 몸이 앞으로 길어지고 정지·착지할 때 세로로 눌립니다.
- 방향은 즉시 좌우 반전하지 않고 몸 폭이 줄었다가 반대편으로 펴집니다.
- 원본 스프라이트의 꼬리 영역과 몸 영역을 겹쳐 분리해 꼬리가 몸보다 늦게
  따라오는 회전으로 그립니다.
- 운반 중에는 몸 중심을 낮추고 꼬리 흔들림과 상하 튀김을 줄입니다.
- 킁킁 탐색 중에는 작은 호흡과 코 확인 리듬만 사용하고 디버그 선이나
  부가 표시는 그리지 않습니다.

## 4. 책임과 호환성

| 구성 | 책임 |
| --- | --- |
| `GatheringMouse` | 행동 상태, 속도, 방향, 보폭, 꼬리와 자세 |
| `WorldView` | 경로와 보이는 작업조 구성 유지 |
| `GameManager` | 생산량·강화·보상 계산 유지 |

- 저장 데이터와 스키마 9는 변경하지 않습니다.
- 외부 서버, 온라인 기능과 새 경제 수치는 추가하지 않습니다.
- 기존 두 쥐 스프라이트만으로 실행 가능해야 합니다.

## 5. 완료 조건

- 쥐가 정지 상태에서 목표 속도까지 점진적으로 가속합니다.
- 경로 중간에서 탐색 상태에 들어갔다가 자동으로 이동을 재개합니다.
- 자원과 굴에서 기존 보상·휴식 왕복이 끊기지 않습니다.
- 방향 반전이 한 프레임 좌우 뒤집기가 아닌 짧은 회전 전환으로 보입니다.
- 운반 여부에 따라 속도와 몸·꼬리 운동량이 달라집니다.
- 서로 다른 시각 인덱스의 쥐가 같은 타이밍으로 탐색하지 않습니다.
- Godot 전체 테스트, 메인 headless 실행, Web/PWA와 프로덕션 빌드가
  통과합니다.
