# V0.2.6 작업 계획 — 시각 성장 정합성

작성일: 2026-07-19
상태: 구현 및 자동 검증 완료 · iPad 실기기 검증 대기

## 1. 작업 목적

V0.2의 주요 기능은 구현되어 있지만 속도·운반·쥐구멍 강화의 외형 단계가 여러
화면 코드에 암묵적으로 흩어져 있고, 다음 보상 진행도는 UI가 게임 규칙을 직접
계산하고 있습니다. V0.2 완료 판정을 내리기 전에 성장 규칙의 책임을 분명히 하고
문서와 실제 구현을 일치시킵니다.

## 2. 이번 범위

1. 속도, 운반, 쥐구멍 레벨을 외형 티어로 바꾸는 단일 규칙을 추가합니다.
2. 이동 쥐에 속도 티어의 목도리·신발과 운반 티어의 가방·전문 운송 표식을
   기본 도형으로 표현합니다.
3. 쥐구멍 보강·저장·조명 표현도 같은 외형 티어 규칙을 사용합니다.
4. 다음 지역 또는 지역 전체 해금 뒤 군락 목표의 진행 정보를 `GameManager`
   API에서 계산합니다.
5. UI는 계산된 진행 정보를 표시만 하며 현재 장비 티어를 함께 안내합니다.
6. 티어 경계, 다음 보상 계산, 대표 모바일 화면 배치를 headless 테스트로
   검증합니다.
7. README, 기준 문서, 기술 문서, 로드맵과 날짜별 changelog를 실제 상태에 맞게
   갱신합니다.

## 3. 제외 범위

- V0.3 지역 선택과 재방문
- 선택지가 있는 위험 사건
- 쥐 종류, 희귀도, 직업과 도감
- 저장 필드 추가 및 스키마 변경
- Web 배포와 iPad 실기기 판정
- 배경 이미지 재제작

## 4. 확정 외형 티어

| 성장축 | 티어 경계 | 표현 |
| --- | --- | --- |
| 속도 | Lv.0 / 1 / 3 / 6 | 기본 / 목도리 / 운동화 / 숙련 질주 |
| 운반 | Lv.0 / 1 / 3 / 6 | 앞발 운반 / 천 주머니 / 배낭 / 전문 운송 |
| 쥐구멍 | Lv.1 / 2 / 3 / 4 / 5 | 작은 틈 / 보강 굴 / 저장 굴 / 마을 입구 / 문명 관문 |

티어는 저장하지 않고 기존 레벨에서 계산합니다. 따라서 저장 스키마는 1을
유지하며 기존 V0.1·V0.2 저장과 호환됩니다.

## 5. 책임과 변경 대상

| 대상 | 책임 |
| --- | --- |
| `scripts/core/visual_progression.gd` | 레벨→외형 티어 및 표시 이름의 단일 기준 |
| `scripts/core/game_manager.gd` | 다음 보상 진행 정보 구성 |
| `scripts/mouse/mouse.gd` | 계산된 티어에 따른 이동 쥐 외형 표현 |
| `scripts/world/world_view.gd` | 계산된 티어에 따른 쥐구멍 표현 |
| `scripts/ui/ui_manager.gd` | 상태 API 결과와 장비 이름 표시 |
| `tests/test_runner.gd` | 경계값, 보상 진행도, 화면 회귀 검증 |

## 6. 완료 조건

- 속도·운반·쥐구멍 티어 경계가 자동 테스트로 고정됩니다.
- 속도와 운반 강화가 각각 최소 3단계의 서로 다른 화면 표현으로 이어집니다.
- 다음 지역 진행도와 전체 지역 해금 뒤 군락 진행도가 0~100 범위로 계산됩니다.
- `GameUI`가 지역 임계값이나 군락 구간을 직접 계산하지 않습니다.
- 1366×1024 가로형과 768×1024 세로형 레이아웃 회귀 테스트가 통과합니다.
- Godot headless import와 전체 테스트, `git diff --check`가 통과합니다.
- 구현 결과와 남은 실기기 검증 항목이 2026-07-19 changelog에 기록됩니다.

## 7. 중단·재검토 기준

- 기존 저장 필드 변경이 필요해지면 구현을 멈추고 스키마 및 마이그레이션 범위를
  다시 정의합니다.
- 기본 도형 장식이 기존 스프라이트를 가려 식별성을 낮추면 장식 크기와 위치만
  조정하고 새 외부 에셋 제작으로 범위를 넓히지 않습니다.
- 자동 테스트로 판정할 수 없는 iPad 실제 가독성은 완료로 추정하지 않고
  `실기기 검증 대기`로 남깁니다.

## 8. 구현 결과

- `VisualProgression`을 추가해 세 성장축의 티어 경계와 표시 이름을 한곳으로
  통합했습니다.
- 이동 쥐에 목도리, 운동화, 숙련 표시와 주머니, 배낭, 전문 운송 표시를
  기본 도형으로 추가했습니다.
- 배경이 있는 지역에서 호출되지 않던 쥐구멍 성장 장식과 생활쥐 그리기를
  실제 화면 흐름에 연결했습니다.
- 다음 지역과 군락 목표 계산을 `GameManager.get_next_reward_summary()`로 옮기고
  UI는 반환된 상태를 표시하도록 정리했습니다.
- 세로형 768×1024와 가로형 1366×1024 UI 회귀 검증을 테스트에 포함했습니다.
- 영구 저장 필드는 변경하지 않았으며 저장 스키마 1을 유지했습니다.
