# V0.4.2 작업 계획 — 첫 영웅 선택

작성일: 2026-07-19
상태: 구현·자동 검증 완료

## 1. 목적

V0.4.1까지 쥐는 역할별 인원 수로만 존재합니다. V0.4.2는 군락의 기존 쥐 중
한 마리를 이름과 이야기가 있는 첫 영웅으로 선택해, 플레이어의 군락에 처음으로
개별 정체성과 영구적인 선택 기록을 만듭니다.

## 2. 사용자 흐름

```text
전체 쥐 5마리 달성
  → 하단 영웅 버튼에서 세 후보 확인
  → 후보 점을 직접 눌러 이름·이야기·효과 미리보기
  → 선택 확정
  → 선택한 영웅이 군락에 남고 실제 효과 적용
```

- 후보 점을 누르는 것만으로 확정하지 않고 별도 `이 영웅 선택` 버튼을 누릅니다.
- V0.4.2에서는 한 번 확정한 영웅을 교체할 수 없습니다.
- 영웅은 기존 인구 한 마리에게 정체성을 부여하므로 `mouse_count`는 변하지
  않습니다.

## 3. 후보와 실제 효과

| ID | 이름·칭호 | 이야기 | 효과 |
| --- | --- | --- | --- |
| `dandani` | 단단이 · 첫 운반대장 | 가장 무거운 치즈를 끝까지 놓지 않은 쥐 | 일반 채집과 예상 생산 +10% |
| `saebyeok` | 새벽 · 틈길 길잡이 | 모두가 잠든 시간에 안전한 길을 찾은 쥐 | 지역 사건·현장 행동 보상 +15% |
| `boreum` | 보름 · 보육실 지킴이 | 어린 점들의 떨림을 가장 먼저 알아챈 쥐 | 돌봄 1회 시간 단축 15초 → 20초 |

- 단단이 효과는 지역 사건·현장 행동의 별도 영웅 보너스로 중복 적용하지
  않습니다. 일반 `collect_trip`과 예상 생산에만 적용합니다.
- 새벽 효과는 탐험쥐 보너스와 곱연산합니다.
- 보름 효과는 새끼당 최대 돌봄 3회 규칙을 바꾸지 않습니다.

## 4. 데이터와 저장 스키마 7

후보 정의는 `data/mice/heroes.json`에서 관리합니다.

새 저장 필드:

```text
selected_hero_id
```

- 스키마 6 이하 저장은 선택한 영웅 없음으로 이관합니다.
- 저장된 ID가 후보 데이터에 없으면 안전하게 선택 없음으로 복구합니다.
- 영웅 해금은 전체 쥐 수로 매번 계산하고 별도 플래그를 저장하지 않습니다.

## 5. 점 기반 표현

- `HeroChoiceView`는 세 후보를 삼각형의 큰 점으로 표시합니다.
- 각 점에 후보 이름을 문자로 표시하고, 선택 후보는 보호 고리와 맥동으로
  강조합니다.
- 작은 기억 점들이 후보 주위를 순환해 기존 점 표현 언어를 유지합니다.
- 마우스와 터치는 같은 좌표 판정 함수를 사용합니다.

## 6. 책임

| 구성 | 책임 |
| --- | --- |
| `data/mice/heroes.json` | 후보 이름, 칭호, 이야기와 효과 설명 |
| `GameManager` | 해금, ID 검증, 영웅 확정, 실제 효과와 저장 상태 |
| `HeroChoiceView` | 후보 점 렌더링, 좌표 판정과 미리보기 선택 신호 |
| `GameUI` | 후보 설명, 확정 버튼과 이미 선택된 영웅 상태 표시 |
| `SaveManager` | 스키마 7 마이그레이션 |

## 7. 제외 범위

- 영웅 교체와 초기화
- 영웅 레벨·유대·장비
- 개별 쥐 전체 명단
- 영웅 전용 스프라이트
- 전투와 영웅 사망
- 서버·온라인 기능
- Cloudflare 독립 배포와 사용자 도메인 변경

## 8. 완료 조건

- 쥐 5마리 미만에서는 영웅 선택이 잠기고 이유가 표시됩니다.
- 후보 점 선택 전에는 확정 버튼이 비활성화됩니다.
- 잘못된 후보 ID와 두 번째 영웅 선택은 거부됩니다.
- 후보 확정 시 전체 쥐 수와 역할 합계는 변하지 않습니다.
- 단단이·새벽·보름의 효과가 각각 지정된 시스템에만 적용됩니다.
- 저장 후 재실행해도 영웅 선택이 유지됩니다.
- 스키마 1~6 저장이 스키마 7로 이관됩니다.
- 세로·가로 대표 화면에서 영웅 패널이 잘리지 않습니다.
- Godot 전체 테스트, Web export, PWA 검증과 프로덕션 빌드가 통과합니다.

## 9. 구현 결과

- 세 후보 정의를 `data/mice/heroes.json`으로 분리하고 잘못된 데이터에는
  기본 후보를 사용하는 복구 경로를 추가했습니다.
- `HeroChoiceView`가 후보 점, 이름, 기억 점 궤도와 마우스·터치 좌표 판정을
  담당합니다.
- 영웅 미리보기와 영구 확정을 분리하고 스키마 7로 선택 ID를 저장합니다.
- 일반 채집, 지역 활동과 보육 돌봄의 영웅 효과 경로를 분리했습니다.
- 빈 저장과 영웅 저장 재실행에서 Godot 전체 테스트를 연속 통과했습니다.
- 메인 씬 headless 실행, Web export, PWA 검사와 Vinext 프로덕션 빌드를
  통과했습니다.
