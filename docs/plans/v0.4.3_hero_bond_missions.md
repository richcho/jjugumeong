# V0.4.3 작업 계획 — 영웅 유대 임무

작성일: 2026-07-19
상태: 구현·자동 검증 완료, 운영 배포 준비

## 1. 목적

V0.4.2의 첫 영웅 선택은 영구 효과를 고른 뒤 추가 참여가 없습니다. V0.4.3은
선택한 영웅마다 다른 점 임무를 반복 수행하고 유대 Lv.1~3을 성장시켜, 이름과
이야기가 지속적인 플레이 관계로 이어지게 합니다.

## 2. 사용자 흐름

```text
첫 영웅 선택
  → 영웅 기록에서 전용 임무 시작
  → 영웅별 규칙에 따라 빛나는 점 5단계 직접 조작
  → 치즈 보상과 유대 Lv. 상승
  → 60초 뒤 다시 임무 수행
  → 유대 Lv.3에서 고유 효과 최대 성장
```

- 임무 중 잘못된 점을 누르면 실수가 1회 기록되고 진행이 한 단계 후퇴합니다.
- 유대 Lv.3 뒤에도 임무를 반복해 기본 치즈 보상을 받을 수 있습니다.
- 앱을 닫은 동안에도 완료 Unix 시각 기준으로 임무 대기 시간이 흐릅니다.

## 3. 영웅별 임무

| 영웅 | 임무 | 조작 |
| --- | --- | --- |
| 단단이 | 흔들리는 짐의 균형 | 좌우로 번갈아 빛나는 짐점 5개 누르기 |
| 새벽 | 어둠 속 틈길 기억 | 굽은 안전 경로의 점 5개를 순서대로 누르기 |
| 보름 | 어린 점의 숨결 맞추기 | 보호 원을 도는 숨결점 5개를 순서대로 누르기 |

모든 임무는 같은 `HeroBondView` 입력·완료 인터페이스를 사용하되 좌표와
시각 구성이 다릅니다.

## 4. 유대 성장과 보상

| 항목 | 값 |
| --- | ---: |
| 유대 최대 레벨 | 3 |
| 임무 단계 | 5 |
| 임무 대기 시간 | 60초 |
| 완료 보상 | 현재 운반량 3회분 |

유대에 따른 고유 효과:

| 영웅 | 선택 직후 | 유대 Lv.1 | Lv.2 | Lv.3 |
| --- | ---: | ---: | ---: | ---: |
| 단단이 일반 채집 | +10% | +12% | +14% | +16% |
| 새벽 지역 활동 | +15% | +20% | +25% | +30% |
| 보름 돌봄 단축 | 20초 | 22초 | 24초 | 26초 |

- 새벽 효과는 탐험쥐 배수와 곱연산합니다.
- 단단이 효과는 일반 채집에만 적용합니다.
- 보름 효과는 이미 적용한 돌봄이나 기존 완료 시각을 소급 변경하지 않습니다.

## 5. 저장 스키마 8

새 필드:

```text
hero_bond_level
next_hero_mission_unix
```

- 스키마 7 이하 저장은 유대 Lv.0, 즉시 임무 가능 상태로 이관합니다.
- 선택 영웅이 없으면 유대 레벨과 대기 시간을 0으로 정리합니다.
- 유대 레벨은 0~3으로 제한하고 미래의 비정상 대기 시각은 일반 Unix 완료
  시각 규칙으로 안전하게 계산합니다.

## 6. 책임

| 구성 | 책임 |
| --- | --- |
| `GameManager` | 임무 검증, 보상, 유대 성장, 대기 시간, 효과 계산과 저장 |
| `HeroBondView` | 영웅별 점 배치, 입력 판정, 실수와 완료 신호 |
| `GameUI` | 선택 화면과 임무 화면 전환, 상태·보상·대기 안내 |
| `SaveManager` | 스키마 8 마이그레이션 |

## 7. 제외 범위

- 영웅 교체·초기화·사망
- 유대 Lv.4 이상
- 장비와 스킬 트리
- 긴 대화 장면과 전용 일러스트
- 임무 자동 완료
- 서버·온라인 기능
- Cloudflare 독립 배포와 사용자 도메인 변경

## 8. 완료 조건

- 영웅 미선택 상태에서는 임무를 시작하거나 완료할 수 없습니다.
- 선택 영웅과 다른 영웅 ID의 완료 요청은 거부됩니다.
- 올바른 점 5단계를 마쳐야 보상과 유대가 증가합니다.
- 잘못된 입력은 실수와 한 단계 후퇴로 반영됩니다.
- 대기 시간 중 반복 완료는 거부됩니다.
- 유대는 정확히 Lv.3에서 멈추고 각 영웅 효과가 표의 값과 일치합니다.
- 저장 후 재실행해도 유대와 대기 시간이 유지됩니다.
- 스키마 1~7 저장이 스키마 8로 이관됩니다.
- 세로·가로 대표 화면에서 영웅 임무가 잘리지 않습니다.
- Godot 전체 테스트, Web export, PWA 검증과 프로덕션 빌드가 통과합니다.

## 9. 구현·검증 결과

- 세 영웅의 서로 다른 점 배치와 오입력 후퇴 규칙을 `HeroBondView`로
  구현했습니다.
- 임무 보상, 60초 대기, 유대 Lv.3 상한과 성장 효과를 `GameManager`에
  연결했습니다.
- 스키마 8 마이그레이션과 저장 왕복을 구현했습니다.
- Godot 4.4.1 전체 테스트를 빈 저장과 저장 재실행에서 연속 통과했습니다.
- 대표 세로·가로 UI 경계, 영웅별 입력, 효과와 대기 시간 회귀 검증을
  통과했습니다.
- 운영 배포 주소와 버전은 배포 완료 뒤 이 문서에 기록합니다.
