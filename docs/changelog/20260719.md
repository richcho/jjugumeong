# 2026-07-19 — V0.2.6 시각 성장 정합성

## 배경과 계획

V0.2 기능은 대부분 구현됐지만 외형 성장의 단계가 화면 코드에 흩어져 있었고,
다음 보상 진행도도 UI가 직접 계산하고 있었습니다. 구현 전에
[`V0.2.6 작업 계획`](../plans/v0.2.6_visual_progression.md)에 범위, 제외 범위,
책임과 완료 조건을 먼저 확정했습니다.

지역 선택, 위험 사건과 직업은 V0.3 범위이므로 이번 작업에 포함하지 않았습니다.
영구 저장 데이터도 추가하지 않아 저장 스키마 1을 유지했습니다.

## 구현

### 외형 티어 단일 기준

- `VisualProgression`을 추가해 속도 Lv.0/1/3/6, 운반 Lv.0/1/3/6,
  쥐구멍 Lv.1/2/3/4/5 경계를 단일 규칙으로 정의
- 속도 강화에 목도리, 운동화와 숙련 질주 표시 추가
- 운반 강화에 천 주머니, 배낭과 전문 운송 표시 추가
- UI에서 현재 속도·운반 장비 이름 표시
- 쥐구멍 보강, 저장 선반과 조명도 같은 티어 규칙 사용

### 보이는 성장 연결 수정

배경이 있는 5개 지역에서 쥐구멍 성장 장식과 생활쥐 그리기 함수가 정의돼
있었지만 실제 그리기 흐름에서 호출되지 않는 상태를 확인했습니다.

- 지역 배경 위에 쥐구멍 성장 장식을 다시 연결
- 보유 쥐 수에 따른 굴 앞 생활쥐 표현을 다시 연결
- 외부 에셋이 없어도 티어 장식은 기본 도형으로 표현

### 다음 보상 책임 분리

- 다음 지역 또는 전체 지역 해금 뒤 군락 목표를
  `GameManager.get_next_reward_summary()`에서 계산
- 제목, 현재값, 목표값, 남은 값과 0~100 진행도를 상태 API로 제공
- `GameUI`에서는 지역 임계값과 군락 구간 계산을 제거하고 결과만 표시

### 회귀 테스트

- 세 성장축의 모든 티어 경계 테스트
- 다음 지역 50% 진행과 군락 목표 50% 진행 테스트
- 기존 768×1024 세로형에 1366×1024 가로형 UI 레이아웃 검증 추가

## V0.3.1 — 지역 도감, 첫 발견 보상과 화면 표시 정리

구현 전에

### 지역 도감과 첫 발견 보상

- 지역 패널을 `지역 도감 · 이동`으로 확장
- 해금 지역별 자원, 위험, 사건 이름, 발견 상태와 현재 지역 표시

- 쥐 구매 수에 따라 굴 앞에 별도로 그리던 장식쥐 제거
- 실제 왕복하는 쥐, 치즈 운반 스프라이트, 쥐 수와 생산 효과는 유지
- 외형 티어 계산과 UI 이름은 후속 전용 아트 연결을 위해 유지

## V0.3.2 — 점 기반 현장 행동

지역 사건의 설명만 다르고 실제 결과가 안전 보상 또는 가속으로 반복되는 문제를
확인했습니다. 구현 전에
[`V0.3.2 작업 계획`](../plans/v0.3.2_dot_actions.md)에 직접 조작, 실패 피드백,
영구 완료 기록과 제외 범위를 확정했습니다.

### 세 가지 직접 행동

- 낡은 부엌 `실타래 속 비밀길`: 실의 연결을 따라 여섯 매듭을 순서대로 누름
- 식료품 창고 `선반 철탑 오르기`: 아래부터 번갈아 놓인 일곱 발판을 밟음
- 동네 편의점 `무한 진열 통로`: 중앙 신호와 같은 색의 문을 다섯 번 선택
- 잘못된 입력은 실수로 기록하고 실타래·철탑 진행을 한 단계 후퇴
- 점, 선, 원과 맥동만 사용하는 공통 `DotActionView` 추가
- 마우스와 터치 입력을 같은 좌표 판정 함수로 처리

### 실제 게임 상태 연결

- 하단 `현장 행동` 버튼과 768×1024 대응 행동 패널 추가
- 완료 전에는 UI가 보상을 직접 지급하지 않도록 `GameManager`에서 행동 ID 검증
- 기본 운반 3회분, 첫 완료 추가 2회분 보상
- 완료 행동을 지역 도감과 통계에 표시
- 반복 행동 공통 20초 대기 시간 적용

### 저장 스키마 3

- `completed_field_action_ids`와 `next_field_action_unix` 추가
- 스키마 2 이하 저장은 완료 행동 없음, 대기 시간 0으로 마이그레이션
- 입력 도중의 임시 단계는 저장하지 않고 완료 보상과 발견 기록을 원자적으로 저장

## V0.4.1 — 군락 역할 보드

보육실에서 늘어난 쥐가 모두 같은 채집 행동만 반복하지 않도록, 구현 전에

효과, 입력 규칙과 저장 구조를 확정했습니다.

- 쥐 3마리부터 채집·탐험·건설의 세 역할점 해금

- 탐험쥐 1마리당 지역 사건·현장 행동 보상 10% 증가, 최대 30%
- 건설쥐 1마리당 이후 새끼 성장 시간 10% 단축, 최대 30%
- 새로 구매하거나 보육실에서 합류한 쥐는 채집 역할에 자동 배치

- 스키마 1~5 저장은 기존 전체 인구를 채집 역할로 이관
- 잘못된 합계는 로드 시 전체 인구와 맞추고 최소 채집쥐를 보장

## V0.4.3 — 영웅 유대 임무

### 영웅별 직접 임무

- 단단이는 좌우 짐점, 새벽은 굽은 틈길점, 보름은 보호 원의 숨결점 5개 조작

- 새벽은 유대마다 지역 활동 +5%p, 최대 +30%
- 보름은 유대마다 돌봄 단축 +2초, 최대 26초
