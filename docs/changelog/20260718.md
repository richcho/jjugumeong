# 2026-07-18 — V0.1 Alpha와 프로젝트 기준 문서 정리

## 배경과 의도

쥐구멍 프로젝트의 첫 실행 가능한 버전을 만들고 iPad Safari에서 직접 확인할 수
있게 했습니다. 초기 문서는 장기 아이디어와 현재 구현이 짧게 섞여 있어, 무엇이
이미 동작하고 다음에 무엇을 만들어야 하는지 구분하기 어려웠습니다.

Alpha Spark 프로젝트의 `현재 기준 문서 + 결정 대기 보드 + 날짜별 변경 기록`
방식을 게임 개발에 맞게 적용했습니다.

## 구현 내용

### 게임 V0.1

- 한 마리의 쥐가 쥐구멍과 치즈 사이를 자동 왕복
- 속도, 운반량, 쥐 수, 쥐구멍 강화
- 누적 치즈 기반 5개 지역 해금과 생산 보너스
- 황금치즈 10초 5배 사건
- 화면 클릭 3초 속도 부스트
- 튜토리얼, 통계와 상태 토스트
- 3초 자동 저장, Web 이중 백업, 백업 복구와 최대 4시간 오프라인 보상

### iPad Web/PWA

- Godot 단일 스레드 Web 내보내기
- Vinext 기반 Sites 호스팅 패키지
- WebAssembly gzip 전송과 서비스 워커 캐시 보정
- 홈 화면 PWA 메타데이터와 가로 방향 설정

### 한글 글꼴 수정

Godot Web의 기본 대체 글꼴이 한국어 glyph를 제공하지 않아 iPad에서 글자가
깨졌습니다.

- Noto Sans KR 서브셋을 프로젝트에 내장
- 프로젝트 기본 테마와 `WorldView` 직접 그리기 글꼴에 적용
- 핵심 UI의 플랫폼 의존 이모지를 한국어 텍스트로 교체
- 한국어 glyph 로드 테스트 추가

### Safari 화면 수정

iPad Safari의 주소창과 동적 화면 높이 때문에 게임 캔버스가 화면 밖으로 커지고
좌측 상단에 걸쳐 보일 수 있었습니다.

- Godot Stretch Aspect를 `expand`로 설정
- 상·하단 UI 안전 여백 확대
- 좁은 화면의 2열 버튼 높이 확보
- `viewport-fit=cover`와 동적 뷰포트 단위 적용
- 캔버스를 현재 보이는 Safari 화면에 고정

## 후속 수정 — Safari 재진입 저장 초기화

Safari에서 화면을 나갔다 다시 들어오면 진행이 초기화되는 사례를 보강했습니다.

- 자동 저장 주기를 10초에서 3초로 단축
- 주요 강화 구매 직후 즉시 저장
- 화면 포커스 이탈과 앱 일시정지 때 즉시 저장
- Web 파일 작성 후 IndexedDB 파일 시스템 동기화 요청
- 같은 JSON을 `localStorage`에도 이중 저장
- 파일 저장과 Web 백업 중 타임스탬프가 최신인 정상본을 로드
- 영구 저장이 허용되지 않은 환경은 UI에 `Safari 저장 제한` 표시
- 소형 쿠키를 세 번째 Web 저장본으로 추가
- 하단에 `수동 저장` 버튼과 확인 토스트 추가
- 화면 제목에 `ALPHA 0.1.2`를 표시해 실행 버전 확인 가능
- 새 서비스 워커 즉시 활성화와 자동 새로고침 적용

## 후속 수정 — Alpha 테스트 속도와 보상 문구

짧은 iPad 테스트에서 지역·강화·저장을 모두 확인할 수 있도록 정식 밸런스와
분리된 `ALPHA_TEST_REWARD_MULTIPLIER`를 추가하고 현재값을 25배로 설정했습니다.
예상 생산량과 오프라인 보상도 같은 계수를 사용합니다.

- 첫 왕복부터 강화 구매를 확인할 수 있도록 기본 보상 가속
- 황금치즈는 Alpha 25배에 사건 5배를 추가 적용
- 상단에 `ALPHA 보상 25배`를 표시
- 황금치즈 상태 문구를 짧게 줄이고 넘침 방지 적용
- 입고 보상 글자의 실제 폭을 계산해 좌우 안전 여백 안으로 제한
- 상단·하단 패널과 겹치지 않도록 보상 글자의 세로 위치 제한

## V0.2 Alpha — 보이는 성장 1차

숫자와 버튼 중심이던 화면을 계획서의 `보이는 성장` 방향으로 재구성했습니다.

- 낡은 부엌, 식료품 창고, 동네 편의점 전용 배경 추가
- 화면을 가리던 예정 기능 버튼을 핵심 화면에서 제거
- 성장 버튼 4개와 통계·저장 보조 행동을 분리
- 다음 지역, 누적 진행도와 시각 보상을 보여 주는 다음 보상 카드 추가
- 지역 해금 발견 카드와 즉시 저장 추가
- 속도 강화 시 목도리, 운반 강화 시 가방 표현
- 쥐구멍 레벨에 따라 보강재, 저장 선반과 조명 추가
- 움직이는 빛 먼지와 쥐 걸음 흔들림으로 생동감 보강
- 지역 배경이 없어도 기본 도형 화면으로 실행되는 폴백 유지
- 대형 식당과 치즈 공장 전용 배경 추가
- 쥐 수가 늘면 굴 앞에서 정리하고 쉬는 생활쥐 추가

## V0.2.1 — 배경 원근과 쥐 화풍 통합

배경의 바닥보다 위에 있던 이동 기준점을 지역 이미지 속 실제 쥐구멍과 자원
위치로 옮겼습니다.

- 5개 배경마다 쥐구멍과 자원 원본 좌표 지정
- 배경 크롭과 같은 공식으로 좌표를 변환해 4:3 iPad에서도 위치 일치
- 이동 차선 간격을 줄여 쥐가 같은 바닥 위를 이동하도록 조정
- 배경 위에 중복으로 그리던 큰 쥐구멍과 치즈 도형 제거
- 실제 배경 오브젝트에는 성장 장식과 부드러운 상태 표시만 추가
- 도형 쥐를 배경 화풍에 맞춘 페인팅 스프라이트로 교체
- 기본 이동형과 치즈 운반형 상태 분리
- 생활쥐도 같은 페인팅 스프라이트 사용

## V0.2.2 — 문구 창과 황금치즈 화면 경계

- 상단 정보와 사건 문구를 반응형 그리드로 변경
- 좁은 화면에서 정보는 2열, 사건 문구는 1열로 자동 배치
- 황금치즈·부스트·다음 지역 문구에 말줄임과 화면 경계 적용
- 토스트를 다음 보상 카드 아래에 배치하고 두 줄 자동 줄바꿈 적용
- 큰 입고 보상은 K·M·B 단위로 줄여 화면 폭 초과 방지
- 입고 보상 문구가 상단 정보·보상 카드·토스트와 겹치지 않도록 안전 영역 확대
- 세로 iPad 황금치즈 문구와 토스트 경계 회귀 테스트 추가

## V0.2.3 — 쥐 보행 애니메이션

- 실제 이동 거리에 따라 보행 주기가 진행되도록 구현
- 보폭에 맞춘 몸의 위아래 이동, 기울기와 미세한 늘어남 추가
- 바닥 접지 그림자와 좌우 발 착지 표시 추가
- 걸음마다 뒤로 퍼지는 작은 먼지 효과 추가
- 치즈 귀환형은 보폭과 반동을 줄여 무게감 표현
- 목적지 도착 순간 짧은 눌림 반동 추가
- 여러 쥐의 보행 시작 주기를 다르게 해 기계적인 동기화 방지
- 자동 테스트에서 이동과 함께 보행 주기가 진행되는지 확인

## V0.2.4 — 전체 폭 정보 패널과 바닥 이동

- 상·하단 패널에 실제 가로 전체 폭 앵커 적용
- 왼쪽 패널이 최소 콘텐츠 너비로 줄어 글자가 잘리던 문제 수정
- 한 마리일 때 수직 차선 위치를 0으로 맞춰 바닥 기준점에 정확히 배치
- 여러 쥐는 바닥에서 최대 ±12px만 분산하고 출발 위치를 가로로 나눔
- 이동 경로 선과 점은 `SHOW_ROUTE_DEBUG`가 켜진 경우에만 표시
- 배포 화면에서는 이동선과 점을 완전히 숨기고 내부 좌표 계산만 유지
- 세로 화면에서 상·하단 패널이 사용 가능 폭을 채우는지 회귀 테스트 추가
