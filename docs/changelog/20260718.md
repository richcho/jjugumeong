# 2026-07-18 — V0.1 Alpha와 프로젝트 기준 문서 정리

## 배경과 의도

쥐구멍 프로젝트의 첫 실행 가능한 버전을 만들고 iPad Safari에서 직접 확인할 수
있게 했습니다. 초기 문서는 장기 아이디어와 현재 구현이 짧게 섞여 있어, 무엇이
이미 동작하고 다음에 무엇을 만들어야 하는지 구분하기 어려웠습니다.

Alpha Spark 프로젝트의 `현재 기준 문서 + 결정 대기 보드 + 날짜별 변경 기록`
방식을 게임 개발에 맞게 적용했습니다.

## 구현 내용

### 게임 V0.1

- 한 마리의 쥐가 쥐구멍과 치즈 사이를 자동 왕복
- 속도, 운반량, 쥐 수, 쥐구멍 강화
- 누적 치즈 기반 5개 지역 해금과 생산 보너스
- 황금치즈 10초 5배 사건
- 화면 클릭 3초 속도 부스트
- 튜토리얼, 통계와 상태 토스트
- 3초 자동 저장, Web 이중 백업, 백업 복구와 최대 4시간 오프라인 보상

### iPad Web/PWA

- Godot 단일 스레드 Web 내보내기
- Vinext 기반 Sites 호스팅 패키지
- WebAssembly gzip 전송과 서비스 워커 캐시 보정
- 홈 화면 PWA 메타데이터와 가로 방향 설정

### 한글 글꼴 수정

Godot Web의 기본 대체 글꼴이 한국어 glyph를 제공하지 않아 iPad에서 글자가
깨졌습니다.

- Noto Sans KR 서브셋을 프로젝트에 내장
- 프로젝트 기본 테마와 `WorldView` 직접 그리기 글꼴에 적용
- 핵심 UI의 플랫폼 의존 이모지를 한국어 텍스트로 교체
- 한국어 glyph 로드 테스트 추가

### Safari 화면 수정

iPad Safari의 주소창과 동적 화면 높이 때문에 게임 캔버스가 화면 밖으로 커지고
좌측 상단에 걸쳐 보일 수 있었습니다.

- Godot Stretch Aspect를 `expand`로 설정
- 상·하단 UI 안전 여백 확대
- 좁은 화면의 2열 버튼 높이 확보
- `viewport-fit=cover`와 동적 뷰포트 단위 적용
- 캔버스를 현재 보이는 Safari 화면에 고정

## 후속 수정 — Safari 재진입 저장 초기화

Safari에서 화면을 나갔다 다시 들어오면 진행이 초기화되는 사례를 보강했습니다.

- 자동 저장 주기를 10초에서 3초로 단축
- 주요 강화 구매 직후 즉시 저장
- 화면 포커스 이탈과 앱 일시정지 때 즉시 저장
- Web 파일 작성 후 IndexedDB 파일 시스템 동기화 요청
- 같은 JSON을 `localStorage`에도 이중 저장
- 파일 저장과 Web 백업 중 타임스탬프가 최신인 정상본을 로드
- 영구 저장이 허용되지 않은 환경은 UI에 `Safari 저장 제한` 표시
- 소형 쿠키를 세 번째 Web 저장본으로 추가
- 하단에 `수동 저장` 버튼과 확인 토스트 추가
- 화면 제목에 `ALPHA 0.1.2`를 표시해 실행 버전 확인 가능
- 새 서비스 워커 즉시 활성화와 자동 새로고침 적용

## 후속 수정 — Alpha 테스트 속도와 보상 문구

짧은 iPad 테스트에서 지역·강화·저장을 모두 확인할 수 있도록 정식 밸런스와
분리된 `ALPHA_TEST_REWARD_MULTIPLIER`를 추가하고 현재값을 25배로 설정했습니다.
예상 생산량과 오프라인 보상도 같은 계수를 사용합니다.

- 첫 왕복부터 강화 구매를 확인할 수 있도록 기본 보상 가속
- 황금치즈는 Alpha 25배에 사건 5배를 추가 적용
- 상단에 `ALPHA 보상 25배`를 표시
- 황금치즈 상태 문구를 짧게 줄이고 넘침 방지 적용
- 입고 보상 글자의 실제 폭을 계산해 좌우 안전 여백 안으로 제한
- 상단·하단 패널과 겹치지 않도록 보상 글자의 세로 위치 제한
