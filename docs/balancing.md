# 밸런스 기준

이 문서는 현재 코드에 적용된 값과 다음 버전에서 검증할 후보를 분리합니다.
실제 값을 바꾸면 코드·데이터와 이 문서를 함께 갱신합니다.

## 1. 현재 적용값

| 항목 | 현재값 | 출처 |
| --- | ---: | --- |

| 탐험쥐 보너스 | 1마리당 지역 활동 보상 +10%, 최대 +30% | `ROLE_BONUS_*` |
| 건설쥐 보너스 | 1마리당 새 새끼 성장 -10%, 최대 -30% | `ROLE_BONUS_*` |
| 첫 영웅 해금 | 쥐 5마리 | `HERO_UNLOCK_MOUSE_COUNT` |
| 단단이 | 일반 채집과 예상 생산 +10% | `HERO_GATHER_MULTIPLIER` |
| 새벽 | 지역 사건·현장 행동 보상 +15% | `HERO_REGION_MULTIPLIER` |
| 보름 | 돌봄 1회 시간 단축 20초 | `HERO_CARE_REDUCTION_SECONDS` |
| 영웅 유대 최대 | Lv.3 | `HERO_MAX_BOND_LEVEL` |
| 영웅 임무 | 점 5단계, 재도전 60초 | `HeroBondView`, `HERO_MISSION_COOLDOWN_SECONDS` |
| 영웅 임무 보상 | 현재 운반량 3회분 | `HERO_MISSION_REWARD_TRIPS` |

유대 Lv.마다 단단이는 일반 채집 효과가 2%p, 새벽은 지역 활동 효과가 5%p,
보름은 돌봄 단축 시간이 2초 증가합니다. 따라서 유대 Lv.3 최대 효과는 각각
16%, 30%, 26초입니다. 최대 유대 뒤에도 임무 보상은 반복 지급합니다.

강화 비용 공식:

```text
비용 = 반올림(기본 비용 × 1.5 ^ 구매 횟수)
```

예상 생산 공식:

```text
왕복 시간 = (예상 편도 거리 × 2) / 현재 이동속도
1회 보상 = 운반량 × 지역 생산 보너스 × Alpha 테스트 보상
초당 생산 = 1회 보상 × 쥐 수 / 왕복 시간
```

황금치즈가 활성화되면 1회 보상에 5배를 적용합니다.
현재 25배 계수는 지역·강화·저장을 짧은 테스트에서 확인하기 위한 Alpha 전용
가속값입니다. 정식 경제 밸런스 확정 전에는 이 상수만 조정하거나 제거하며,
지역 해금값과 강화 비용의 기준 데이터는 그대로 유지합니다.

## 2. 현재 지역값

`data/stages/stages.json`이 실행 기준입니다.

| 지역 | 해금 누적 치즈 | 생산 보너스 |
| --- | ---: | ---: |
| 낡은 부엌 | 0 | 1.0배 |
| 식료품 창고 | 100 | 1.15배 |
| 동네 편의점 | 500 | 1.35배 |
| 대형 식당 | 2,000 | 1.65배 |
| 치즈 공장 | 10,000 | 2.0배 |

스테이지 보너스는 치즈가 실제로 쥐구멍에 입고될 때 적용합니다. 현재 지역은
누적 치즈 기준으로 자동 갱신되며, 사용한 치즈 때문에 다시 잠기지 않습니다.

## 3. V0.2 보상 간격 원칙

상태: 다음, 정확한 수치는 플레이 테스트 후 확정

- 첫 유의미한 강화는 1~3분 안에 경험
- 지역 사이에는 최소 2개의 시각적 중간 보상 배치
- 다음 보상까지 남은 예상 시간이 10분을 넘으면 작은 목표 추가
- 지역 해금은 생산 배수만이 아니라 배경과 이야기 보상을 함께 제공
- 강화 비용 상승보다 새로운 표현이 늦게 열리지 않게 조정

중간 이정표 후보:

| 지역 구간 | 약 30% | 약 60% | 100% |
| --- | --- | --- | --- |
| 부엌 → 창고 | 첫 장비 | 굴 보강 | 식료품 창고 |
| 창고 → 편의점 | 저장 선반 | 전문 운반쥐 | 동네 편의점 |
| 편의점 → 식당 | 네온 소품 | 원정대 표식 | 대형 식당 |
| 식당 → 공장 | 직업 소품 | 운송 수레 | 치즈 공장 |

위 수치는 UI와 외형 시스템 구현 후 실제 플레이 시간으로 보정합니다.

## 4. 관찰 지표

V0.2부터 다음 항목을 수동 플레이 테스트 표에 기록합니다.

- 첫 강화까지 걸린 시간
- 각 지역 해금까지 걸린 시간
- 버튼별 구매 비율
- 다음 목표를 이해하지 못한 구간
- 화면을 보고 성장했다고 느낀 순간
- 오프라인 보상이 다음 결정을 대신해 버리는지 여부
- iPad에서 잘못 누르거나 읽기 어려운 UI

초기 Alpha에서는 정밀 분석 시스템을 먼저 만들지 않고, 5~10회의 직접 플레이
기록으로 큰 문제를 찾습니다.

## 5. 밸런스 변경 규칙

- 한 번에 비용, 생산량, 이동속도를 모두 바꾸지 않습니다.
- 변경 전후의 지역 도달 시간을 비교합니다.
- 단일 플레이 사례만으로 장기 배율을 크게 바꾸지 않습니다.
- 느린 구간을 황금치즈 확률만 올려 숨기지 않습니다.
- 시각적 보상이 부족한 문제와 경제가 느린 문제를 구분합니다.
- 수치를 바꾸면 changelog에 이유, 이전값, 새값과 검증을 기록합니다.
