# 게임 디자인

이 문서는 플레이어가 무엇을 하고 어떤 보상을 느끼는지 정의합니다. 전체 상태와
우선순위는 `docs/jjugumeong_master_plan.md`, 정확한 수치는
`docs/balancing.md`를 기준으로 합니다.

## 1. 게임의 정체성

쥐구멍은 한 마리의 작은 노동을 관찰하는 즐거움과, 그 노동이 마을과 문명으로
커지는 장기 성취를 연결한 2D 방치형 성장 게임입니다.

핵심 판타지:

> 처음에는 벽의 작은 틈이었지만, 내가 모은 치즈와 동료들로 살아 있는 지하
> 문명이 되었다.

주요 플레이 감정:

- 쥐들이 계속 왕복하는 생동감
- 귀환할 때마다 얻는 짧고 명확한 보상
- 가까운 다음 목표를 보는 기대감
- 강화가 실제 외형 변화로 이어지는 성장감
- 새 지역, 쥐, 사건과 영웅을 발견하는 수집욕

## 2. 핵심 루프

### 2.1 현재 루프

```text
자동 이동 → 치즈 획득 → 귀환·입고 → 강화 구매
    ↑                                ↓
    └──── 더 빠르고 많은 생산 ────────┘
```

플레이어의 능동 행동은 강화 선택과 화면 클릭 부스트입니다. 게임을 보고 있지
않을 때도 최대 4시간의 생산이 계산됩니다.

### 2.2 확장 루프

```text
수집 → 강화 → 시각 변화 → 지역 발견
→ 사건 대응 → 건설·직업 배치 → 연구
→ 탐험·무역 → 문명 선택과 역사
```

새 시스템은 이 순서를 강화해야 하며, 별도의 메뉴에서 숫자만 늘리는 고립된
기능이 되지 않게 합니다.

## 3. 플레이 세션 구조

| 시간 | 플레이어 경험 | 대표 보상 |
| --- | --- | --- |
| 5~20초 | 쥐의 첫 왕복 이해 | 입고 숫자와 효과 |
| 1~3분 | 첫 강화 선택 | 속도 또는 운반 외형 |
| 3~8분 | 여러 쥐와 굴 확장 | 주민과 배경 소품 |
| 10~20분 | 새 지역 목표 달성 | 발견 연출과 지역 세트 |
| 재접속 | 쉬는 동안의 성과 확인 | 오프라인 보상 |
| 장기 | 문명 단계 달성 | 건물, 직업, 지도와 이야기 |

정확한 시간은 플레이 테스트로 조정하며, 다음 보상이 10분 이상 보이지 않는
구간을 만들지 않는 것이 초기 원칙입니다.

## 4. 성장축과 시각 피드백

### 4.1 속도

| 확정 단계 | 기능 | 표현 |
| --- | --- | --- |
| Lv.0 | 기본 이동 | 평범한 집쥐 |
| Lv.1~2 | 빠른 이동 | 목도리 |
| Lv.3~5 | 숙련 이동 | 작은 운동화 |
| Lv.6+ | 고속 이동 | 먼지와 잔상 |

### 4.2 운반

| 확정 단계 | 기능 | 표현 |
| --- | --- | --- |
| Lv.0 | 치즈 1개 | 앞발로 운반 |
| Lv.1~2 | 운반량 증가 | 천 주머니 |
| Lv.3~5 | 대량 운반 | 배낭 |
| Lv.6+ | 전문 운송 | 작은 수레 |

### 4.3 쥐 수

구매한 쥐는 화면에 실제로 등장해야 합니다. 이동 중인 쥐 외에도 굴 앞에서
쉬거나 자원을 정리하는 쥐를 보여 주어, 인구가 숫자가 아니라 공동체로 느껴지게
합니다.

### 4.4 쥐구멍

| 확정 레벨 | 외형 | 이야기 의미 |
| --- | --- | --- |
| Lv.1 | 갈라진 벽의 작은 틈 | 혼자 숨는 곳 |
| Lv.2 | 나뭇가지로 보강한 굴 | 안전한 집 |
| Lv.3 | 치즈 선반이 있는 저장 굴 | 공동 저장소 |
| Lv.4 | 표지판과 터널이 있는 입구 | 지하 마을 |
| Lv.5+ | 조명과 운송로가 있는 관문 | 문명의 중심 |

V0.2.6에서 위 레벨 구간을 외형 티어의 기준으로 확정했습니다. 티어는 저장하지
않고 현재 강화 레벨에서 계산합니다. V0.3.1에서는 쥐의 등·꼬리와 배경 속
쥐구멍 위에 덧그린 도형이 내부 표시처럼 보인다는 피드백에 따라 해당 절차적
표현을 제거했습니다. 단계 구분은 유지하되 다시 연결할 때는 전용 스프라이트나
배경 아트를 사용합니다.

## 5. 넥스트 보상 시스템

현재의 “누적 수량 남음” 문구를 다음 보상 카드로 발전시킵니다.

필수 정보:

- 보상 이름
- 현재 수치와 목표 수치
- 진행 막대
- 해금되는 시각 요소와 기능 미리보기
- 지역 사이의 중간 이정표

보상 예시:

```text
다음 보상: 동네 편의점
342 / 500 치즈

네온 배경 · 과자 자원
배낭 운반쥐 · 야간 폐기 사건
```

지역 달성 시:

1. 기존 생산을 멈추지 않습니다.
2. “새 지역 발견” 카드를 2~3초 보여 줍니다.
3. 배경과 자원이 바뀝니다.
4. 새 위험과 다음 목표를 한 줄로 안내합니다.
5. 저장을 즉시 수행합니다.

## 6. 지역 디자인

### 6.1 낡은 부엌

- 역할: 조작과 자동 왕복을 배우는 시작 지역
- 색감: 어두운 자주색과 따뜻한 치즈색
- 주요 오브젝트: 갈라진 벽, 바닥 몰딩, 냉장고 그림자
- 이야기: 이름 없는 집쥐의 첫 생존
- 보상: 첫 동료와 보강된 굴

### 6.2 식료품 창고

- 역할: 저장과 인구 증가를 체감
- 색감: 푸른 회색, 나무와 마대색
- 주요 오브젝트: 선반, 곡물 자루, 상자
- 위험: 쥐덫
- 사건: 대형 음식물 발견
- 보상: 주머니 운반쥐와 저장 선반

### 6.3 동네 편의점

- 역할: 인간 세계로의 첫 원정
- 색감: 청록색 밤, 분홍·파랑 네온
- 주요 오브젝트: 냉장 진열대, 과자 봉지, CCTV 빛
- 위험: 인간의 발과 청소 도구
- 사건: 야간 폐기 시간
- 보상: 배낭쥐, 편의점 배경, 원정대 정체성

### 6.4 대형 식당

- 역할: 직업과 전문 생산의 예고
- 색감: 따뜻한 주황과 짙은 갈색
- 주요 오브젝트: 조리대, 화로, 접시와 식재료
- 위험: 화재
- 사건: 빵 축제
- 보상: 요리사쥐, 수레 운반, 고급 치즈

### 6.5 치즈 공장

- 역할: 생존 단계의 결산과 자동화 시대 예고
- 색감: 황금색, 철제 회색, 경고등
- 주요 오브젝트: 컨베이어, 배관, 숙성실
- 위험: 기계
- 사건: 황금치즈 발견
- 보상: 기술쥐, 기계식 관문, 연구 시스템 예고

## 7. 사건과 위험 설계 원칙

좋은 위험은 준비와 선택을 만듭니다.

- 나쁜 예: 이유 없이 치즈 20% 손실
- 좋은 예: 고양이 접근 경고 후 `숨기 / 빠르게 운반 / 미끼 사용` 선택
- 좋은 예: 야간 폐기 시간 동안 생산량이 증가하지만 인간 발견 위험도 증가

초기 사건은 한 화면에서 이해할 수 있는 두 선택지와 짧은 결과로 시작합니다.
전투, 장비, 치료가 준비되기 전에는 영구 손실을 크게 주지 않습니다.

### 7.1 V0.3.0 첫 지역 사건

- 해금 지역은 `지역` 패널에서 재방문할 수 있습니다.
- `지역 사건`은 현재 방문 지역의 고유 사건을 엽니다.
- 각 사건은 정확히 두 선택지를 제공합니다.
- 안전 대응은 두 번의 운반량에 해당하는 즉시 보상을 줍니다.
- 빠른 대응은 한 번의 운반 보상과 8초 이동 가속을 줍니다.
- 사건 쿨다운은 전체 지역 공통 30초이며 재실행 뒤에도 유지됩니다.
- 한 번 해결한 사건 ID는 발견 기록에 남습니다.

### 7.2 V0.3.1 지역 도감과 첫 발견

- `지역 도감 · 이동`은 해금 지역의 자원, 위험, 사건과 발견 상태를 함께
  표시합니다.
- 도감 진행도는 `발견 사건 수 / 해금 지역 수`로 계산합니다.
- 각 지역 사건의 첫 해결에는 현재 운반량 3회분을 추가 지급합니다.
- 이미 발견한 사건은 기본 선택 보상만 지급하며 추가 보상을 반복하지 않습니다.
- 발견과 첫 보상 판정은 저장 스키마 2의 `completed_region_event_ids`를
  함께 사용합니다.

### 7.3 V0.3.2 점 기반 현장 행동

선택 문구를 누르는 사건과 별도로, 사용자가 점을 직접 조작해 상황을 해결하는
`현장 행동`을 둡니다.

| 지역 | 행동 | 조작 |
| --- | --- | --- |
| 낡은 부엌 | 실타래 속 비밀길 | 실의 연결 순서를 따라 매듭점 누르기 |
| 식료품 창고 | 선반 철탑 오르기 | 아래에서 위로 번갈아 놓인 발판점 누르기 |
| 동네 편의점 | 무한 진열 통로 | 중앙 신호와 같은 색의 원형 문 찾기 |

- 올바른 입력을 끝까지 수행해야 보상이 확정됩니다.
- 잘못 누르면 실타래와 철탑은 한 단계 후퇴하며 모든 행동은 실수를 기록합니다.
- 첫 완료는 기본 운반 3회분에 2회분을 추가하고, 반복은 기본 보상만 지급합니다.
- 행동 완료 ID와 20초 대기 시간을 저장 스키마 3에 기록합니다.
- 식당과 공장은 이 수직 슬라이스의 실기기 조작감을 확인한 뒤 서로 다른 행동으로
  확장합니다.

### 7.4 V0.4.0 새끼쥐 보육실

- 쥐구멍 Lv.10에서 치즈 500으로 보육실을 건설합니다.
- 보육실 Lv.1은 새끼 2마리를 동시에 보호합니다.
- 새끼는 기본 120초 뒤 성장하며, 작은 새끼 점을 누르는 돌봄 1회마다 15초를
  단축합니다. 한 새끼에는 최대 3회 적용합니다.
- 성장 완료는 자동 인구 증가가 아닙니다. 사용자가 `성체 합류`를 눌러야
  `mouse_count`가 1 증가하고 채집조가 다시 구성됩니다.
- 보호 원, 점 크기, 맥동과 문자를 함께 사용해 색만으로 상태를 구분하지 않습니다.
- 완료 Unix 시각을 저장해 앱 종료 중에도 성장합니다.

## 8. 수집과 장기 성장

장기 수집 대상:

- 쥐 종과 희귀도
- 직업과 고유 특성
- 지역 도감과 자원
- 영웅과 유물
- 플레이어 문명의 역사 기록

세대교체가 추가되더라도 도감, 영웅, 유물과 DNA 같은 정체성 자산은 유지하는
방향을 기본으로 합니다. 정확한 초기화 범위는 경제 시스템이 준비된 뒤 결정합니다.

## 9. UX 원칙

- 다음 행동과 다음 보상은 같은 화면에서 확인할 수 있어야 합니다.
- 버튼 이름은 결과와 비용을 동시에 설명합니다.
- 모바일 터치 버튼은 작게 줄이지 말고 열 수나 정보량을 조정합니다.
- 색상만으로 상태를 구분하지 않고 문자와 형태를 함께 사용합니다.
- 해금 연출은 성취감을 주되 자동 생산을 방해하지 않습니다.
- 예정 기능 버튼은 핵심 화면을 혼잡하게 만들면 숨깁니다.

# 중간 콘텐츠와 쥐 사회

- 지역 해금 이후에는 `세계 소식`이 12초 간격으로 순환해 같은 배경에서도 새로운 상황을 전달한다.
- 각 지역은 탐험 사건과 생활 소식을 함께 가진다. 사건은 위험·통로·교역을, 생활은 식사·휴식·가족·교육을 다룬다.
- 쥐구멍 레벨은 첫 보금자리 → 작은 군락 → 치즈 마을 → 쥐구멍 도시 → 지하 연맹으로 이어진다.
- 지역 전체 해금 뒤의 다음 목표는 공동 식탁, 보육실, 지하 시장, 통로 의회, 심층 탐사 순으로 제시한다.
- 화면의 쥐는 기본 1마리, 최대 3개 채집 조로 제한하고 나머지는 군락 내부에서 생활하는 것으로 표현한다.
- 보유 쥐가 3마리를 넘으면 각 대표 채집 조에 인원을 균등 배분하고 운반 보상을 비율로 합산한다.
- 속도 강화는 제곱근 성장 곡선을 사용한다. 초반 강화는 분명하지만 고레벨일수록 증가폭이 줄어든다.
- 일반 이동은 245, 클릭 가속은 285를 넘지 않아 생물적인 보행 속도를 유지한다.
